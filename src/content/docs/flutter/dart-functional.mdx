# Dart 集合与函数式

集合（List / Set / Map）与"在集合上做变换"的函数式 API（map / where / fold ...）是 Dart 日常写代码的两条主轴。本章把它们一次讲透——这之后看 Flutter 任何业务代码都不会再卡在"这是个什么 API"。

## 三大核心集合

### `List<T>`：有序、可重复

```dart
List<int> a = [1, 2, 3];               // 字面量，类型推断
final b = <int>[];                     // 空列表必须指定泛型
final c = List<int>.filled(5, 0);      // 长度 5，全 0
final d = List<int>.generate(5, (i) => i * i);   // [0,1,4,9,16]
final e = List<int>.of([1, 2, 3]);     // 拷贝
final f = List<int>.unmodifiable([1, 2]);        // 不可变包装
```

**固定长度 vs 可变长度**：

```dart
final g = List<int>.filled(5, 0, growable: false);   // 不能 add / remove
g.add(1);    // 运行时抛错
```

默认 `[]`、`List.of`、`List.from`、`List.generate` 都是**可变**的；`filled` 的 `growable` 默认为 `true`。

### `Set<T>`：无序、不重复

```dart
Set<int> s = {1, 2, 3};
final empty = <int>{};                 // 空 Set 必须显式泛型，否则会被推为 Map
s.add(2);                              // false（已存在）
s.add(4);                              // true
s.contains(3);                         // true
```

底层默认是 `LinkedHashSet`——**保持插入顺序**。需要排序集合用 `SplayTreeSet`。

### `Map<K, V>`：键值对

```dart
Map<String, int> m = {'a': 1, 'b': 2};
m['c'] = 3;
m.remove('a');
m['x'];                                // 类型 int? —— 不存在时返回 null
m.containsKey('b');                    // true

m.putIfAbsent('d', () => 0);           // 不存在则插入并返回新值；存在则返回旧值
m.update('a', (old) => old + 1, ifAbsent: () => 0);  // 一站式更新
```

底层默认是 `LinkedHashMap`——**保持插入顺序**。

### 常量集合

```dart
const list = [1, 2, 3];
const set = {1, 2, 3};
const map = {'a': 1};

list.add(4);       // 运行时 UnsupportedError
```

### 集合字面量里的 `if` / `for` / `...`

第 2 章已经预告过，这里给完整版本：

```dart
final base = [1, 2, 3];
final extra = null;

final all = [
  0,
  ...base,                      // 展开
  ...?extra,                    // 可空展开（null 时跳过）
  if (DateTime.now().isUtc) 4,  // 条件元素
  for (final x in base) x * 10, // 循环展开
  for (var i = 0; i < 3; i++) -i,
];
```

这套语法在构造 widget 列表时无处不在。

## 泛型

### 基础

```dart
T identity<T>(T x) => x;

class Box<T> {
  Box(this.value);
  T value;
}

final b = Box<int>(42);
final b2 = Box(42);   // 推断为 Box<int>
```

### 泛型约束

```dart
T max<T extends Comparable<T>>(T a, T b) => a.compareTo(b) >= 0 ? a : b;

max(1, 2);                              // ✓
// max(Object(), Object());             // ✗ Object 没实现 Comparable
```

### 协变与逆变（一笔带过）

Dart 的泛型默认是**协变**的：`List<Dog>` 能被赋值给 `List<Animal>`。这与 Java 不同（Java 默认不变）。

```dart
class Animal {}
class Dog extends Animal {}

List<Dog> dogs = [Dog()];
List<Animal> animals = dogs;   // ✓ 协变
// animals.add(Cat());          // 编译期 OK，但运行时若 dogs 实际类型是 List<Dog> 会抛 TypeError
```

协变带来便利，也带来"运行时类型不安全"的可能。需要严格不变时用 `covariant` 关键字或限制类型层级。

## `Iterable<T>`：所有惰性序列的共同祖先

`List` 与 `Set` 都实现了 `Iterable`。`Iterable` 的核心方法都是**惰性求值**——在你实际遍历它之前不会执行。

```dart
final result = [1, 2, 3, 4, 5]
    .where((x) {
      print('filter $x');
      return x.isEven;
    })
    .map((x) {
      print('map $x');
      return x * 2;
    });

print('before iterate');
for (final x in result) print('got $x');
```

输出：

```text
before iterate
filter 1
filter 2
map 2
got 4
filter 3
filter 4
map 4
got 8
filter 5
```

注意：

- `where` / `map` 调用时**不执行任何过滤或映射**；
- 真正的执行发生在 `for-in` 内，按需逐元素求值；
- 同一个 `Iterable` 可能被遍历多次——**每次都重新计算**。

### 惰性 → 急切：`.toList()` / `.toSet()`

```dart
final list = result.toList();   // 立刻计算并固化为 List
```

工程经验：**链式调用末尾几乎都该 `.toList()`**——除非你确定后续只迭代一次。否则要么白算几次，要么遇到副作用被多次执行。

## `Iterable` 高频 API

### 变换

```dart
[1, 2, 3].map((x) => x * 2);                    // (2, 4, 6)
[1, 2, 3].expand((x) => [x, x * 10]);            // (1, 10, 2, 20, 3, 30) 一对多展开
[1, 2, 3].mapIndexed((i, x) => '$i:$x');         // 来自 collection 包
```

### 过滤

```dart
[1, 2, 3, 4].where((x) => x.isEven);             // (2, 4)
[1, 'a', 2, 'b'].whereType<int>();               // (1, 2) —— 类型过滤
```

### 折叠

```dart
[1, 2, 3].reduce((a, b) => a + b);               // 6 —— 不带初值，需非空
[1, 2, 3].fold(100, (acc, x) => acc + x);        // 106 —— 带初值
[1, 2, 3].fold<List<String>>([], (acc, x) => [...acc, '$x']);   // ['1','2','3']
```

`reduce` 与 `fold` 区别：`reduce` 用第一个元素当初值，集合不能为空；`fold` 显式提供初值，类型可与元素不同。

### 查询

```dart
final list = [1, 2, 3];
list.first;          // 1（空抛异常）
list.last;           // 3
list.firstOrNull;    // 1（空返回 null，Dart 3 起内置）
list.single;         // 仅有 1 个元素时返回，否则抛
list.elementAt(1);   // 2
list.indexOf(2);     // 1
list.contains(2);    // true
list.any((x) => x > 2);    // true
list.every((x) => x > 0);  // true
list.length;
list.isEmpty;
list.isNotEmpty;
```

### 排序与反转

```dart
final l = [3, 1, 2];
l.sort();                              // 原地排序，返回 void
l.sort((a, b) => b.compareTo(a));      // 自定义比较器（逆序）

[1, 2, 3].reversed;                    // (3, 2, 1) Iterable
```

### 切片与拼接

```dart
final l = [1, 2, 3, 4, 5];
l.sublist(1, 3);              // [2, 3]
l.take(2);                    // (1, 2)
l.skip(2);                    // (3, 4, 5)
l.takeWhile((x) => x < 4);    // (1, 2, 3)
l.skipWhile((x) => x < 3);    // (3, 4, 5)
l.followedBy([6, 7]);         // (1..7) 不复制
[1, 2].toSet().intersection({2, 3});   // {2}
```

### 分组与去重

```dart
final l = [1, 2, 2, 3];
l.toSet().toList();          // [1, 2, 3] 去重（顺序可能改）

// groupBy 不在标准库，需 collection 包
import 'package:collection/collection.dart';

final groups = groupBy(['apple', 'avocado', 'banana'], (s) => s[0]);
// {'a': ['apple', 'avocado'], 'b': ['banana']}
```

`collection` 包是 Dart 官方维护的"补丁"，提供 `firstWhereOrNull`、`groupBy`、`mapIndexed`、`equality`（深比较）等。**业务项目几乎一定会装**。

## Lambda 与函数式风格

### 单表达式 Lambda

```dart
final double_ = (int x) => x * 2;
final names = users.map((u) => u.name).toList();
```

### 多语句 Lambda

```dart
final formatted = users.map((u) {
  final base = u.name.toUpperCase();
  return '$base (${u.age})';
}).toList();
```

### 方法引用（撕掉式调用）

```dart
final lengths = ['a', 'bb', 'ccc'].map((s) => s.length).toList();
// 也可写成：
final lengths2 = ['a', 'bb', 'ccc'].map(_length).toList();
int _length(String s) => s.length;
```

Dart 没有 Java/C# 的 `String::length` 简洁语法，但可以传顶层函数 / 静态方法 / 实例方法的引用。

### 闭包捕获

```dart
List<int Function()> makeCounters(int n) {
  final list = <int Function()>[];
  for (var i = 0; i < n; i++) {
    var x = i;                           // 必须在循环内重新绑定
    list.add(() => x);
  }
  return list;
}

makeCounters(3).map((f) => f()).toList();   // [0, 1, 2]
```

**陷阱**：在循环里直接捕获循环变量（不重新绑定）会让所有闭包共享同一个变量。Dart 的 `for (var i in ...)` 默认每轮重新绑定，但 `for (var i = 0; ...)` 老式写法**不会**——务必显式 `var x = i`。

## 生成器函数：`sync*` / `async*`

Dart 提供两种生成器，分别产生 `Iterable<T>` 与 `Stream<T>`。

### `sync*` 同步生成器

```dart
Iterable<int> naturals() sync* {
  var n = 1;
  while (true) {
    yield n;
    n++;
  }
}

naturals().take(5).toList();     // [1, 2, 3, 4, 5]
```

要点：

- 函数返回类型必须是 `Iterable<T>`；
- 函数体用 `yield` 产出元素；
- `yield*` 可委托给另一个 `Iterable`；
- 完全惰性——`take(5)` 只会拉 5 次。

### `async*` 异步生成器

```dart
Stream<int> ticker(Duration period) async* {
  var n = 0;
  while (true) {
    await Future.delayed(period);
    yield n++;
  }
}

await for (final n in ticker(const Duration(seconds: 1)).take(3)) {
  print(n);     // 0, 1, 2，每秒一个
}
```

返回类型必须是 `Stream<T>`，详见 [Dart 异步与并发](/flutter/dart-async)。

## 扩展方法：让 `Iterable` 用得更顺手

> 上一章已经讲过 `extension` 的语法。这里给几个 Flutter 项目里常见的实用扩展。

```dart
extension IterableX<T> on Iterable<T> {
  T? get firstOrNull => isEmpty ? null : first;
  T? get lastOrNull => isEmpty ? null : last;

  Iterable<T> separatedBy(T separator) sync* {
    final iter = iterator;
    if (!iter.moveNext()) return;
    yield iter.current;
    while (iter.moveNext()) {
      yield separator;
      yield iter.current;
    }
  }
}

['a', 'b', 'c'].separatedBy(',').toList();   // ['a', ',', 'b', ',', 'c']
```

> Dart 3 起 `firstOrNull` / `lastOrNull` / `singleOrNull` / `elementAtOrNull` 已**内置**到 `Iterable`，自己写时改名避免冲突。

## 集合与等价：`==` 默认不深比

```dart
final a = [1, 2, 3];
final b = [1, 2, 3];
print(a == b);   // false ❗ —— List 默认是引用相等
```

要深比较：

```dart
import 'package:collection/collection.dart';

print(const ListEquality().equals(a, b));   // true
print(const DeepCollectionEquality().equals(
  {'a': [1, 2]},
  {'a': [1, 2]},
));   // true
```

集合相等是**新手最常踩的坑之一**。Flutter 项目里凡涉及"对比两个集合是否一致"，**永远显式调用 ListEquality / SetEquality / MapEquality**，或用 `freezed`（自动深比）。

## `const` 集合的特殊行为

```dart
const a = [1, 2, 3];
const b = [1, 2, 3];
print(identical(a, b));   // true ✓ 编译期 canonicalize
```

编译期相同字面量的 `const` 集合**会被合并为同一对象**——这与 `const` 构造的 widget 一样是 Flutter 性能优化的常用招数。

## 性能与抉择

### `List` 选型

- **元素数量已知**：`List.filled(n, v)` 比 `[]..add()` 快得多。
- **大量随机访问 + 偶尔末端追加**：默认 `List<T>`（背后是动态数组）即可。
- **大量在头部插入**：用 `Queue<T>`（双端队列）。
- **永不变更**：`List.unmodifiable` 或 `const`。

### `Map` 选型

- 默认 `LinkedHashMap` 适合 99% 场景。
- 需要按键有序：`SplayTreeMap` 或 `SortedMap`（包）。
- 需要并发安全：Dart 没有 `ConcurrentHashMap`——单 isolate 不需要，跨 isolate 用消息传递。

### 何时不要链式

链式 API 优雅，但每步都生成新 Iterable，调试时栈帧噪音多。性能敏感的循环里，老式 `for-in` + 临时变量更直观也更可调。

## 一个综合示例

把本章主要 API 落到一个统计场景：

```dart filename="lib/stats.dart"
import 'package:collection/collection.dart';

class Order {
  Order(this.user, this.amount);
  final String user;
  final double amount;
}

void main() {
  final orders = [
    Order('alice', 10),
    Order('bob', 20),
    Order('alice', 15),
    Order('alice', 5),
    Order('bob', 30),
  ];

  // 1. 按用户分组
  final grouped = groupBy(orders, (o) => o.user);
  // { 'alice': [Order, Order, Order], 'bob': [Order, Order] }

  // 2. 每个用户求总额
  final totals = grouped.map(
    (user, list) => MapEntry(user, list.fold<double>(0, (s, o) => s + o.amount)),
  );
  // { 'alice': 30.0, 'bob': 50.0 }

  // 3. 按总额降序排序
  final sorted = totals.entries.sorted((a, b) => b.value.compareTo(a.value));

  // 4. 打印 Top 1
  final top = sorted.firstOrNull;
  if (top != null) print('${top.key} spent ${top.value}');
}
```

读懂这段代码意味着你已经掌握本章绝大部分核心。

## Dart 集合与函数式心智模型清单

1. **`List` / `Set` / `Map` 的字面量分别是 `[]` / `{}` / `{:}`**——空 `{}` 推断为 Map，需要 Set 时显式 `<int>{}`。
2. **`Iterable` 默认惰性**：链式调用直到 `toList()` / `toSet()` / `for-in` 才真正执行。
3. **`Map[]` 返回 `V?`**——必须显式处理 null。
4. **集合 `==` 默认是引用相等**——深比要用 `package:collection`。
5. **`for / if / ... / ...?` 在集合字面量里直接可用**，是 Flutter 写 widget 列表的核心语法。
6. **`groupBy` / `firstWhereOrNull` 不在 SDK，要 `package:collection`**。
7. **`sync*` / `async*` 是惰性生成器**——计算昂贵的序列只有真正消费时才跑。
8. **`extension` 扩展集合 API 是 Flutter 项目的常见做法**——但注意 Dart 3 已内置 `firstOrNull` 等。
9. **`const` 集合会被编译期合并为同一对象**，与 `const` 构造的 widget 共同支撑性能。
10. **链式很爽但调试不友好**——性能关键路径回归 `for-in` + 临时变量。
