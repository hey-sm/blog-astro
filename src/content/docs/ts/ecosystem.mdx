# 生态系统工具

TypeScript 的强大不仅来自语言本身，更来自围绕它的工具生态系统。

## ESLint + typescript-eslint

### 安装

```bash
pnpm add -D eslint @eslint/js typescript-eslint
```

### 配置（Flat Config, ESLint 9+）

```ts filename="eslint.config.mts"
import tseslint from 'typescript-eslint'
import js from '@eslint/js'

export default tseslint.config(
  js.configs.recommended,
  ...tseslint.configs.recommendedTypeChecked,
  {
    languageOptions: {
      parserOptions: {
        projectService: true,
        tsconfigRootDir: import.meta.dirname
      }
    }
  },
  {
    rules: {
      '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
      '@typescript-eslint/no-explicit-any': 'warn',
      '@typescript-eslint/consistent-type-imports': ['error', { prefer: 'type-imports' }],
      '@typescript-eslint/array-type': ['error', { default: 'array' }]
    }
  }
)
```

### 为 ESLint 配置 tsconfig 检查范围

```json filename="tsconfig.eslint.json"
{
  "extends": "./tsconfig.json",
  "include": ["src", "eslint.config.mts", "vite.config.ts"]
}
```

### React 项目的 ESLint 规则

```ts
rules: {
  '@typescript-eslint/consistent-type-imports': 'error',
  '@typescript-eslint/no-misused-promises': ['error', {
    checksVoidReturn: { attributes: false }
  }],
  '@typescript-eslint/no-unnecessary-condition': 'error',
  '@typescript-eslint/strict-boolean-expressions': 'error'
}
```

## Prettier

```json filename=".prettierrc"
{
  "semi": false,
  "singleQuote": true,
  "trailingComma": "all",
  "printWidth": 100,
  "tabWidth": 2
```

```bash
pnpm add -D prettier eslint-config-prettier
```

```ts filename="eslint.config.mts"
import prettier from 'eslint-config-prettier'

export default tseslint.config(
  // ... 其他配置
  prettier  // 放在最后，覆盖之前的格式化规则
)
```

```json filename="package.json"
{
  "scripts": {
    "lint": "eslint .",
    "format": "prettier --write .",
    "format:check": "prettier --check ."
  }
}
```

### tsx（推荐——ESM + 即时执行）

```bash
pnpm add -D tsx

# 运行

# Watch 模式
pnpm tsx --watch src/server.ts
```

### ts-node

```bash
pnpm add -D ts-node

pnpm ts-node src/script.ts
```

## CI/CD 中的 TypeScript

### GitHub Actions

```yaml filename=".github/workflows/ci.yml"
jobs:
  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm typecheck
      - run: pnpm lint
```

```ts filename="tsup.config.ts"
import { defineConfig } from 'tsup'

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['esm', 'cjs'],
  dts: true,     // 生成 .d.ts
  clean: true,
  sourcemap: true
```

## Source Map 调试

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "sourceMap": true,
    "declaration": true,
    "declarationMap": true
  }
}
```

- `sourceMap`: 运行时调试可映射回 `.ts` 源码
- `declarationMap`: 在 IDE 中跳转定义时直达 `.ts` 源文件而非 `.d.ts`

## 陷阱

### 1. 跳过 skipLibCheck 会隐藏真实的类型错误

```json
// ❌ 尽量不要依赖此选项
{ "compilerOptions": { "skipLibCheck": true } }

// ✅ 如果确实因为第三方类型导致构建失败，
// 升级 @types 版本而非跳过检查
```

### 2. @types 版本与库版本不一致

```bash
# ❌ 危险：@types/react 版本过旧导致缺新类型
pnpm list @types/react
# 检查是否与 react 主版本一致
```

### 3. ts-node vs tsx 的 ESM/CJS 差异

```bash
# ts-node 默认 CJS，配置 ESM 复杂
# tsx 原生支持 ESM，推荐用于新项目
```

## 最佳实践

### ✅ CI 中运行 typecheck + lint 双检查

```yaml
- run: pnpm typecheck && pnpm lint
```

### ✅ 使用 tsc --noEmit 进行快速类型检查

```bash
pnpm exec tsc --noEmit  # 最快（跳过增量信息写入）
```

### ✅ 库项目生成 declaration + declarationMap

```json
{ "compilerOptions": { "declaration": true, "declarationMap": true } }
```
