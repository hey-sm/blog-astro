# 数据类型

ECMAScript 将值分为两大类：

- **原始类型（primitive types）**：`String` / `Number` / `BigInt` / `Boolean` / `Undefined` / `Null` / `Symbol`，共 7 种
- **对象类型（Object）**：函数、数组、`Date`、`RegExp`、用户定义的类实例等，统一归为 `Object`

原始类型的值是**不可变（immutable）**且**按值传递（pass by value）**；对象类型则是**可变**且**按引用传递（pass by reference）**。

## 原始类型

### `String`

字符串是 UTF-16 编码的字符序列。

```js
const a = 'single'
const b = "double"
const c = `template ${1 + 1}`                 // 模板字符串支持插值与多行

console.log(a.length)                         // → 6
console.log('a' + 'b')                        // → 'ab'   一律返回新字符串

const s = 'hello'
s[0] = 'H'                                    // 静默失败（严格模式抛错）
console.log(s)                                // → 'hello'   字符串不可变
```

字符串的"按下标修改"会被忽略；任何看似修改的操作都会返回新字符串。

#### Unicode 与码点

```js
'😀'.length                                    // → 2   该字符占两个 UTF-16 码元（surrogate pair）
[...'😀'].length                               // → 1   迭代器以"码点"为单位
'😀'.codePointAt(0)                           // → 128512
String.fromCodePoint(128512)                  // → '😀'
```

处理 emoji、罕见汉字、组合字符时，必须使用基于码点的 API（如展开运算符、`for...of`、`String.prototype.codePointAt`），而非 `.length` 与 `[i]`。

### `Number`

JavaScript 中的所有数字（不含 `BigInt`）都是 IEEE 754 双精度浮点。

```js
typeof 42                                     // → 'number'
typeof 3.14                                   // → 'number'
typeof Infinity                               // → 'number'
typeof NaN                                    // → 'number'

Number.MAX_SAFE_INTEGER                       // → 9007199254740991  即 2^53 - 1
0.1 + 0.2 === 0.3                             // → false   浮点精度问题
Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON   // → true    工程上的相等判定

Number.isNaN(NaN)                             // → true
NaN === NaN                                   // → false   NaN 不等于自身
```

#### 进制字面量

```js
0b1010                                        // → 10    二进制
0o755                                         // → 493   八进制
0xff                                          // → 255   十六进制
1_000_000                                     // → 1000000   数字分隔符（ES2021）
```

### `BigInt`

ES2020 引入，用于表示超出 `Number.MAX_SAFE_INTEGER` 范围的整数。

```js
const big = 9007199254740993n                 // 字面量后缀 n
const big2 = BigInt('9007199254740993')

big + 1n                                      // → 9007199254740994n
big + 1                                       // ✗ TypeError: Cannot mix BigInt and other types
Number(big)                                   // → 9007199254740992   精度可能丢失
```

`BigInt` 与 `Number` 不能直接运算，需先显式转换。

### `Boolean`

仅有 `true` / `false` 两个值。常被自动从其他类型转换得到（详见[类型转换](/js/type-casting)）：

```js
Boolean(0)                                    // → false
Boolean('')                                   // → false
Boolean(null)                                 // → false
Boolean(undefined)                            // → false
Boolean(NaN)                                  // → false

// 上述 6 个值（含 false 自身）称为 falsy 值；其余皆为 truthy
Boolean({})                                   // → true   空对象是 truthy
Boolean([])                                   // → true   空数组是 truthy
```

### `Undefined`

表示**变量已声明、尚未赋值**或**函数未返回值**：

```js
let x
console.log(x)                                // → undefined

function f() {}
console.log(f())                              // → undefined

const obj = {}
console.log(obj.missing)                      // → undefined   不存在的属性
```

### `Null`

表示**有意为之的"无"**。语义上区别于 `undefined`：`undefined` 是"未定义"，`null` 是"故意置空"：

```js
let cache = null                              // 显式表达"暂无缓存"

typeof null                                   // → 'object'   规范层面的历史遗留 Bug，无法修复
null === null                                 // → true
null == undefined                             // → true       == 下二者宽松相等
null === undefined                            // → false
```

### `Symbol`

ES6 引入的唯一标识符，常用于对象属性键，避免命名冲突：

```js
const a = Symbol('desc')
const b = Symbol('desc')
console.log(a === b)                          // → false   每次调用生成全新唯一值

const KEY = Symbol('private')
const obj = { [KEY]: 'hidden' }

Object.keys(obj)                              // → []          Symbol 键不会被 keys 枚举
Object.getOwnPropertySymbols(obj)             // → [Symbol(private)]

// 全局注册：跨 realm 共享
const s1 = Symbol.for('shared')
const s2 = Symbol.for('shared')
console.log(s1 === s2)                        // → true
```

引擎内部使用 `Symbol.iterator`、`Symbol.asyncIterator`、`Symbol.toPrimitive` 等 well-known symbols 暴露元行为：

```js
const range = {
  from: 1,
  to: 5,
  [Symbol.iterator]() {
    let cur = this.from
    return {
      next: () => cur <= this.to
        ? { value: cur++, done: false }
        : { value: undefined, done: true }
    }
  }
}

for (const n of range) console.log(n)         // → 1, 2, 3, 4, 5
```

## 对象类型

除上述 7 种原始类型外，所有值都是 `Object`。对象的本质是**键值对集合 + 内部槽**。

```js
const user = {
  name: 'Ada',
  age: 36,
  greet() { return `hi, ${this.name}` }
}

user.name                                     // → 'Ada'
user['name']                                  // → 'Ada'   等价访问

user.email = 'ada@example.com'                // 动态添加
delete user.age                                // 删除属性
'age' in user                                 // → false
```

### 属性描述符

每个属性背后都有一个**属性描述符（property descriptor）**：

```js
const obj = {}
Object.defineProperty(obj, 'name', {
  value: 'js',
  writable: false,        // 是否可写
  enumerable: false,      // 是否可枚举（for...in / Object.keys）
  configurable: false     // 是否可重新配置或删除
})

obj.name = 'ts'                               // 静默失败
delete obj.name                               // 静默失败
console.log(Object.getOwnPropertyDescriptor(obj, 'name'))
// → { value: 'js', writable: false, enumerable: false, configurable: false }
```

普通字面量赋值产生的属性默认 `writable / enumerable / configurable` 全为 `true`。

### 访问器属性

属性也可以被定义为 getter / setter：

```js
const o = {
  _value: 0,
  get value() { return this._value },
  set value(v) {
    if (typeof v !== 'number') throw new TypeError()
    this._value = v
  }
}

o.value = 42
console.log(o.value)                          // → 42
o.value = 'bad'                               // ✗ TypeError
```

### 原型与原型继承

每个对象都有一个内部槽 `[[Prototype]]`，指向另一个对象（或 `null`）。访问属性时若自身没有，会沿原型链向上查找。

```js
const animal = {
  eat() { console.log(`${this.name} is eating`) }
}

const dog = Object.create(animal)
dog.name = 'Rex'
dog.eat()                                     // → 'Rex is eating'

Object.getPrototypeOf(dog) === animal         // → true
```

`class` 语法是原型继承的语法糖：

```js
class Animal {
  constructor(name) { this.name = name }
  eat() { console.log(`${this.name} is eating`) }
}

class Dog extends Animal {
  bark() { console.log(`${this.name}: woof!`) }
}

const d = new Dog('Rex')
d.eat()                                       // → 'Rex is eating'
d.bark()                                      // → 'Rex: woof!'

Object.getPrototypeOf(d) === Dog.prototype    // → true
Object.getPrototypeOf(Dog.prototype) === Animal.prototype // → true
```

引擎查找属性的算法对应规范的 `[[Get]]` / `[[GetPrototypeOf]]` 抽象操作：

```text
访问 d.eat
  → d 自身有 eat? 没有
  → 查 Object.getPrototypeOf(d)（= Dog.prototype）有 eat? 没有
  → 查 Dog.prototype 的原型（= Animal.prototype）有 eat? 是
  → 返回该方法
```

### 内置对象

| 类别 | 代表 |
| ---- | ---- |
| 包装对象 | `String` / `Number` / `Boolean` / `Symbol` / `BigInt` |
| 索引集合 | `Array` / `TypedArray`（`Int8Array` 等）/ `ArrayBuffer` |
| 键控集合 | `Map` / `Set` / `WeakMap` / `WeakSet` |
| 时间 | `Date` |
| 文本 | `RegExp` |
| 错误 | `Error` / `TypeError` / `RangeError` / `SyntaxError` 等 |
| 反射 | `Reflect` / `Proxy` |
| 异步 | `Promise` / `AsyncFunction` |

详见 [数据结构](/js/data-structures) 与 [异步](/js/async)。

## `typeof` 操作符

`typeof` 返回一个表示类型的字符串。其行为有几处反直觉：

```js
typeof undefined                              // → 'undefined'
typeof null                                   // → 'object'        ← 历史遗留
typeof 1                                      // → 'number'
typeof 1n                                     // → 'bigint'
typeof 'a'                                    // → 'string'
typeof true                                   // → 'boolean'
typeof Symbol()                               // → 'symbol'
typeof {}                                     // → 'object'
typeof []                                     // → 'object'        ← 数组也是对象
typeof function(){}                           // → 'function'      ← 唯一被单列的对象子类
typeof undeclared                             // → 'undefined'     ← 未声明也不抛错
```

### 精准判别值的类型

```js
// 1. Array.isArray
Array.isArray([])                             // → true

// 2. Object.prototype.toString.call —— 通用方案
Object.prototype.toString.call([])            // → '[object Array]'
Object.prototype.toString.call(null)          // → '[object Null]'
Object.prototype.toString.call(/re/)          // → '[object RegExp]'
Object.prototype.toString.call(new Date())    // → '[object Date]'

// 3. instanceof —— 检查原型链
[] instanceof Array                           // → true
new Date() instanceof Date                    // → true
```

`instanceof` 的局限：跨 frame / iframe / vm context 时，`Array.prototype` 不是同一个对象，导致误判。生产代码中应优先使用 `Array.isArray` 等专用 API。

## 值类型 vs 引用类型

```js
// 原始类型按值复制
let a = 1
let b = a
b = 2
console.log(a)                                // → 1

// 对象按引用共享
let o1 = { x: 1 }
let o2 = o1
o2.x = 99
console.log(o1.x)                             // → 99   o1 与 o2 引用同一对象
```

#### 函数参数同样遵循该规则

```js
function modify(obj) { obj.x = 100 }
const target = { x: 1 }
modify(target)
console.log(target.x)                         // → 100   引用被修改

function reassign(obj) { obj = { x: 100 } }
const target2 = { x: 1 }
reassign(target2)
console.log(target2.x)                        // → 1     形参重新指向新对象，不影响实参
```

JavaScript 的"引用传递"实质是**按值传递引用**（pass-by-sharing）。

## 小结

- 7 种原始类型 + 1 种对象类型，是所有 JS 值的全部分类；
- 原始类型不可变、按值传递；对象类型可变、按引用共享；
- `typeof null === 'object'` 是规范遗留 Bug，需要使用 `Object.prototype.toString` 或专用 API 精准判别；
- `class` 是原型继承的语法糖，访问属性时沿 `[[Prototype]]` 链查找。
