## V8 引擎工作原理与垃圾回收

### V8 执行流程

1. **解析（Parsing）** → 生成 AST（抽象语法树）
2. **解释执行（Ignition）** → 生成字节码并执行
3. **JIT 编译（TurboFan）** → 热代码优化为机器码

### 垃圾回收机制

**分代回收策略**：将堆内存分为新生代和老生代。

#### 新生代（Scavenger 算法）

采用 **复制算法**，将内存分为 From-Space 和 To-Space：

1. 对象首先分配到 From-Space
2. GC 时，存活对象复制到 To-Space
3. From-Space 被清空
4. 交换 From-Space 和 To-Space

**对象晋升条件**：
- 经历过一次 Scavenge 仍存活
- To-Space 使用超过 25%

```js
// 新生代对象示例
function createShortLived() {
  const temp = { data: new Array(1000) };
  return temp.data.length; // temp 很快被回收
}
```

#### 老生代（Mark-Sweep-Compact）

- **标记阶段**：从根对象遍历，标记所有可达对象
- **清除阶段**：回收未标记对象
- **整理阶段**：移动对象，消除内存碎片

```js
// 老生代对象示例
const cache = {}; // 全局变量，长期存活

function addToCache(key, value) {
  cache[key] = value; // 会晋升到老生代
}
```

**1. 闭包引用分离的 DOM 节点**

```js

  const onClick = () => {
    console.log(element.id);
  };
  
  // 即使从 DOM 移除，element 仍被闭包引用

```

```js

    // data 无法被回收
```

**3. 意外的全局变量**

```js

```

## 事件循环（Event Loop）

### 浏览器事件循环

执行顺序：**同步代码 → 微任务（Microtask）→ 宏任务（Macrotask）→ 渲染**

```js
console.log('1. 同步');

setTimeout(() => console.log('4. 宏任务'), 0);

Promise.resolve().then(() => console.log('3. 微任务'));

console.log('2. 同步');

// 输出：1 → 2 → 3 → 4
```

### Node.js 事件循环

六个阶段（按顺序执行）：

1. **timers**：执行 `setTimeout` / `setInterval` 回调
2. **pending callbacks**：执行延迟到下一轮的 I/O 回调
3. **idle, prepare**：内部使用
4. **poll**：检索新的 I/O 事件
5. **check**：执行 `setImmediate` 回调
6. **close callbacks**：执行关闭回调（如 `socket.on('close')`）

```js
// Node.js 执行顺序示例
setTimeout(() => console.log('setTimeout'), 0);
setImmediate(() => console.log('setImmediate'));

process.nextTick(() => console.log('nextTick'));

Promise.resolve().then(() => console.log('Promise'));

// 输出：nextTick → Promise → setTimeout/setImmediate（顺序不定）
```

**`process.nextTick()` 优先级最高**：在当前阶段结束后、下一阶段开始前立即执行。

```js
setImmediate(() => console.log('setImmediate'));

process.nextTick(() => {
  console.log('nextTick');
  process.nextTick(() => console.log('nextTick 嵌套'));
});

// 输出：nextTick → nextTick 嵌套 → setImmediate
```

## 高级性能优化

### Tree Shaking

基于 ES Modules 的静态分析，消除未使用的代码。

```js
// utils.js
export function used() {
  return 'I am used';
}

export function unused() {
  return 'I will be removed';
}

// main.js
import { used } from './utils.js';
console.log(used());

// 打包后 unused() 被移除
```

**注意事项**：
- 仅支持 ES Modules（`import`/`export`）
- 避免副作用代码（在 `package.json` 中标记 `"sideEffects": false`）

## 内存管理与诊断

### 内存泄漏常见原因

**1. 全局变量累积**

```js
// ❌ 持续累积
let globalCache = [];
function addData(data) {
  globalCache.push(data); // 永不清理
}

// ✅ 使用 WeakMap
const cache = new WeakMap();
function addData(obj, data) {
  cache.set(obj, data); // obj 被 GC 后，缓存自动清除
}
```

**2. 遗忘的事件监听器**

```js
// ❌ 未清理
function setupListener() {
  const button = document.getElementById('btn');
  button.addEventListener('click', handleClick);
  // 忘记 removeEventListener
}

// ✅ 清理监听器
function setupListener() {
  const button = document.getElementById('btn');
  const controller = new AbortController();
  
  button.addEventListener('click', handleClick, {
    signal: controller.signal
  });
  
  // 清理
  return () => controller.abort();
}
```

## 软件设计模式

### 单例模式（Singleton）

确保类只有一个实例，并提供全局访问点。

```js
class Singleton {
  constructor() {
    if (Singleton.instance) {
      return Singleton.instance;
    }
    Singleton.instance = this;
    this.data = {};
  }
  
  getData() {
    return this.data;
  }
}

const instance1 = new Singleton();
const instance2 = new Singleton();
console.log(instance1 === instance2); // true
```

### 观察者模式（Observer）

对象间一对多依赖关系，当主体状态改变时，所有依赖者收到通知。

```js
class Subject {
  constructor() {
    this.observers = [];
  }
  
  attach(observer) {
    this.observers.push(observer);
  }
  
  detach(observer) {
    this.observers = this.observers.filter(obs => obs !== observer);
  }
  
  notify(data) {
    this.observers.forEach(observer => observer.update(data));
  }
}

class Observer {
  constructor(name) {
    this.name = name;
  }
  
  update(data) {
    console.log(`${this.name} received:`, data);
  }
}

// 使用
const subject = new Subject();
const observer1 = new Observer('Observer1');
const observer2 = new Observer('Observer2');

subject.attach(observer1);
subject.attach(observer2);
subject.notify('Event occurred!');
```

### 发布-订阅模式（Pub-Sub）

观察者模式的解耦版本，通过事件中心通信。

```js
class EventBus {
  constructor() {
    this.events = {};
  }
  
  on(event, callback) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
  }
  
  emit(event, data) {
    if (this.events[event]) {
      this.events[event].forEach(callback => callback(data));
    }
  }
  
  off(event, callback) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter(cb => cb !== callback);
    }
  }
}

// 使用
const bus = new EventBus();

bus.on('userLogin', (user) => {
  console.log('Analytics:', user);
});

bus.on('userLogin', (user) => {
  console.log('UI Update:', user);
});

bus.emit('userLogin', { name: 'Alice', id: 123 });
```

### 工厂模式（Factory）

根据输入创建不同类型的对象。

```js
class Car {
  constructor(type) {
    this.type = type;
  }
  drive() {
    console.log(`Driving a ${this.type}`);
  }
}

class CarFactory {
  static createCar(type) {
    switch (type) {
      case 'sedan':
        return new Car('Sedan');
      case 'suv':
        return new Car('SUV');
      default:
        throw new Error('Unknown car type');
    }
  }
}

const sedan = CarFactory.createCar('sedan');
sedan.drive(); // 'Driving a Sedan'
```

### 代理模式（Proxy）

为对象提供代理，控制对原对象的访问。

```js
const target = {
  name: 'Alice',
  age: 30
};

const handler = {
  get(obj, prop) {
    console.log(`Getting ${prop}`);
    return obj[prop];
  },
  set(obj, prop, value) {
    console.log(`Setting ${prop} to ${value}`);
    if (prop === 'age' && typeof value !== 'number') {
      throw new TypeError('Age must be a number');
    }
    obj[prop] = value;
    return true;
  }
};

const proxy = new Proxy(target, handler);
console.log(proxy.name);   // Getting name → 'Alice'
proxy.age = 31;             // Setting age to 31
// proxy.age = '31';        // TypeError
```

## Web 安全

### XSS（跨站脚本攻击）

**类型**：
- **存储型**：恶意脚本存储在服务器（如评论）
- **反射型**：通过 URL 参数注入
- **DOM 型**：前端 JS 直接操作 DOM

**防御措施**：

```js
// ❌ 危险：直接插入 HTML
function renderComment(comment) {
  document.getElementById('comments').innerHTML = comment;
  // 若 comment = '<script>alert("XSS")</script>'，会被执行
}

// ✅ 安全：使用 textContent
function renderComment(comment) {
  const div = document.createElement('div');
  div.textContent = comment; // 自动转义
  document.getElementById('comments').appendChild(div);
}

// ✅ 安全：输出编码
function escapeHtml(text) {
  const map = {
    '<': '&lt;',
    '>': '&gt;',
    '&': '&amp;',
    '"': '&quot;',
    "'": '&#x27;'
  };
  return text.replace(/[<>&"']/g, char => map[char]);
}
```

**CSP（内容安全策略）**：

```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; script-src 'self' https://trusted.cdn.com">
```

### CSRF（跨站请求伪造）

**攻击原理**：利用用户已登录状态，诱导发起恶意请求。

**防御措施**：

**1. Anti-CSRF Token**

```js
// 后端生成 Token
app.get('/form', (req, res) => {
  const token = generateToken();
  req.session.csrfToken = token;
  res.render('form', { csrfToken: token });
});

// 前端提交时携带
<form method="POST" action="/transfer">
  <input type="hidden" name="csrf_token" value="<%= csrfToken %>">
  <button>Submit</button>
</form>

// 后端验证
app.post('/transfer', (req, res) => {
  if (req.body.csrf_token !== req.session.csrfToken) {
    return res.status(403).send('Invalid CSRF token');
  }
  // 处理请求
});
```

**2. SameSite Cookie**

```js
res.cookie('sessionId', token, {
  httpOnly: true,
  sameSite: 'strict' // 或 'lax'
});
```

## 框架原理（React）

### Virtual DOM

React 在内存中维护 DOM 树的轻量表示，避免直接操作真实 DOM。

**工作流程**：
1. 状态改变 → 生成新 Virtual DOM
2. Diff 算法对比新旧 Virtual DOM
3. 计算最小更新集
4. 批量更新真实 DOM

### Diffing 算法

**三大策略**：

**1. Tree Diff**：只对比同层节点，不跨层对比

```jsx
// 移动节点会被视为删除 + 新增
<div>
  <A />
</div>

// 变为
<div>
  <B />
  <A />
</div>
// React 会删除 <A />，创建 <B /> 和新 <A />
```

**2. Component Diff**：
- 同类型组件继续对比子节点
- 不同类型组件直接替换

**3. Element Diff**：使用 `key` 优化列表

```jsx
// ❌ 使用 index 作 key
{items.map((item, index) => (
  <li key={index}>{item}</li>
))}
// 插入头部时，所有节点 key 都变了，全部重新渲染

// ✅ 使用唯一 ID
{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}
```

```ts
type IsString<T> = T extends string ? 'yes' : 'no';

// 实用示例：提取返回值类型
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
```

**背压处理**：

```js
const readable = fs.createReadStream('large-file.txt');

  const canContinue = writable.write(chunk);
  if (!canContinue) {
    readable.pause(); // 暂停读取

```

### 多进程

利用多核 CPU，提升性能。

```js
const { fork } = require('child_process');

// 主进程
const child = fork('worker.js');

child.send({ task: 'compute', data: [1, 2, 3] });

child.on('message', (result) => {
  console.log('Result:', result);
});

// worker.js
process.on('message', (msg) => {
  if (msg.task === 'compute') {
    const result = msg.data.reduce((a, b) => a + b, 0);
    process.send({ result });
  }
});
```

**Cluster 模式**：

```js
const cluster = require('cluster');
const http = require('http');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);
  
  // 创建工作进程
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker) => {
    console.log(`Worker ${worker.process.pid} died`);
    cluster.fork(); // 重启
  });
} else {
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('Hello from worker ' + process.pid);
  }).listen(8000);
  
  console.log(`Worker ${process.pid} started`);
}
```
