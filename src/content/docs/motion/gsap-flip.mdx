# Flip

`Flip` 插件让任意 DOM 结构 / 样式变化**平滑过渡**——和 Motion 的 `layout` prop 一样，底层都是 FLIP 算法（First, Last, Invert, Play）。

## 与 Motion `layout` 的差异

| 维度 | Motion `layout` | GSAP Flip |
| ---- | --------------- | --------- |
| 触发方式 | 声明式（React state 变） | 命令式（手动调用 capture / from） |
| 与 React 生命周期 | 自动 | 需要在 effect 里手动协调 |
| 控制粒度 | 中等 | 高（可指定哪些属性变化） |
| 跨容器移动 | `layoutId` 支持 | ⭐ 天然支持 |
| 复杂时序 | 受限 | 配合 timeline 任意编排 |

**React 项目里大多数布局动画用 Motion `layout` 更省事**；GSAP Flip 适合**已经用 GSAP 主导动画**或**精细控制**的场景。

## 安装

```tsx
import gsap from 'gsap'
import { Flip } from 'gsap/Flip'

gsap.registerPlugin(Flip)
```

## 核心 API：3 步用法

Flip 永远是这三步：

1. **`Flip.getState(targets)`** — 记录元素**当前**状态（位置、尺寸、旋转、透明度）
2. **改变 DOM**（React state 变、加 class、移动节点）
3. **`Flip.from(state, options)`** — 比对前后差异、播放过渡

```tsx filename="components/ToggleLayout.tsx"
import { useGSAP } from '@gsap/react'
import gsap from 'gsap'
import { Flip } from 'gsap/Flip'
import { useRef, useState } from 'react'

gsap.registerPlugin(Flip)

export function ToggleLayout() {
  const container = useRef<HTMLDivElement>(null)
  const [expanded, setExpanded] = useState(false)

  useGSAP(
    () => {
      // 在状态变化前抓取
      const state = Flip.getState('.card')

      // (此时 React 已经 re-render 完，DOM 是新状态)
      Flip.from(state, {
        duration: 0.6,
        ease: 'power2.inOut',
      })
    },
    { dependencies: [expanded], scope: container }
  )

  return (
    <div ref={container}>
      <button onClick={() => setExpanded(!expanded)}>Toggle</button>
      <div className={expanded ? 'grid grid-cols-1' : 'grid grid-cols-3'}>
        <div className="card">A</div>
        <div className="card">B</div>
        <div className="card">C</div>
      </div>
    </div>
  )
}
```

> ⚠️ 上面写法有问题——`Flip.getState` 在 `useGSAP` 重跑时已经是新状态了。**正确做法**是用 `useLayoutEffect` 在 React 提交前后两个时机分别拍快照。

### 正确的 React 集成

由于 React 重新渲染后 DOM 已经是新状态，需要"事先快照"。最优雅是用 `useLayoutEffect`：

```tsx filename="components/ToggleLayout.tsx"
import { useLayoutEffect, useRef, useState } from 'react'
import gsap from 'gsap'
import { Flip } from 'gsap/Flip'

gsap.registerPlugin(Flip)

export function ToggleLayout() {
  const containerRef = useRef<HTMLDivElement>(null)
  const stateRef = useRef<Flip.FlipState | null>(null)
  const [expanded, setExpanded] = useState(false)

  // 1. React 重新渲染前抓状态（在 setState 调用前）
  const handleToggle = () => {
    stateRef.current = Flip.getState('.card')
    setExpanded(e => !e)
  }

  // 2. 重新渲染后播放过渡
  useLayoutEffect(() => {
    if (!stateRef.current) return
    Flip.from(stateRef.current, { duration: 0.6, ease: 'power2.inOut' })
    stateRef.current = null
  })

  return (
    <div ref={containerRef}>
      <button onClick={handleToggle}>Toggle</button>
      <div className={expanded ? 'grid grid-cols-1' : 'grid grid-cols-3'}>
        <div className="card">A</div>
        <div className="card">B</div>
        <div className="card">C</div>
      </div>
    </div>
  )
}
```

模式总结：

- 在 **事件回调** 里 `getState` 后再 `setState`
- 在 **`useLayoutEffect`** 里 `Flip.from(...)`

## Flip.from 配置

```tsx
Flip.from(state, {
  duration: 0.7,
  ease: 'power2.inOut',
  stagger: 0.05,
  absolute: true,         // 临时设为 absolute 避免触发其他 layout
  scale: true,            // 用 scale 实现尺寸变化（比 width/height 流畅）
  fade: true,             // 加入 / 移除元素加渐隐
  spin: 1,                // 跨容器移动时旋转一圈
  toggleClass: 'flipping', // 动画中给元素加 class
  onComplete: () => console.log('done'),
})
```

### scale: true

用 scale 而不是 width/height 实现尺寸变化——避免 layout 重排，性能好得多。

### absolute: true

布局动画期间把元素临时设为 `position: absolute`——避免移动元素影响其他元素的 layout 流。

### fade: true

如果 `state` 之后元素列表有增删，新元素淡入、移除元素淡出。

## Flip.fit

`fit` 不动画，只是**计算"把 A 放到 B 的位置"需要什么 transform**：

```tsx
Flip.fit(elementA, elementB, { scale: true })
```

适合"先快速把 A 摆到 B 位置，再单独做其他动画"的场景。

## 跨容器移动

Flip 的杀手锏——把元素从一个容器搬到另一个容器，动画自动平滑：

```tsx
const handleMove = () => {
  const state = Flip.getState('.item')

  // 把 .item 移动到新容器
  newParent.appendChild(itemElement)

  Flip.from(state, { duration: 0.7, ease: 'power2.inOut' })
}
```

React 版本：

```tsx
const [inLeft, setInLeft] = useState(true)

const handleMove = () => {
  stateRef.current = Flip.getState('.item')
  setInLeft(!inLeft)
}

useLayoutEffect(() => {
  if (stateRef.current) {
    Flip.from(stateRef.current, { duration: 0.7 })
    stateRef.current = null
  }
})

return (
  <>
    <div>{inLeft && <div className="item">Move me</div>}</div>
    <div>{!inLeft && <div className="item">Move me</div>}</div>
  </>
)
```

元素从一个 div 飞到另一个 div——丝滑过渡。

## 与 Motion `layoutId` 的对比

Motion 的 `layoutId` 实现同样效果：

```tsx
{inLeft
  ? <motion.div layoutId="item">Move</motion.div>
  : null
}
{!inLeft
  ? <motion.div layoutId="item">Move</motion.div>
  : null
}
```

声明式更省事——React 项目里**这种场景优先用 Motion**。

## targets 选项

只对部分变化做动画：

```tsx
const state = Flip.getState('.card', { props: 'opacity, backgroundColor' })

Flip.from(state)  // 只动画 opacity 和 backgroundColor
```

`props` 指定额外要追踪的属性（位置 / 尺寸 / 旋转默认追踪）。

## 配合 timeline

Flip 返回一个 timeline——能像普通 timeline 一样控制：

```tsx
const tl = Flip.from(state, { duration: 0.7 })

tl.add(() => console.log('flip 完成'))
tl.timeScale(2)
tl.reverse()
```

也可以把 Flip 作为大 timeline 的一部分：

```tsx
const master = gsap.timeline()
master.add(Flip.from(state, { duration: 0.5 }))
       .to('.next', { opacity: 1, duration: 0.3 })
```

## 实战：列表排序

```tsx filename="components/SortableList.tsx"
import { useLayoutEffect, useRef, useState } from 'react'
import gsap from 'gsap'
import { Flip } from 'gsap/Flip'

gsap.registerPlugin(Flip)

const initial = [
  { id: 1, label: 'Apple' },
  { id: 2, label: 'Banana' },
  { id: 3, label: 'Cherry' },
]

export function SortableList() {
  const [items, setItems] = useState(initial)
  const stateRef = useRef<Flip.FlipState | null>(null)

  const shuffle = () => {
    stateRef.current = Flip.getState('.item')
    setItems(prev => [...prev].sort(() => Math.random() - 0.5))
  }

  useLayoutEffect(() => {
    if (stateRef.current) {
      Flip.from(stateRef.current, {
        duration: 0.5,
        ease: 'power2.inOut',
        stagger: 0.05,
      })
      stateRef.current = null
    }
  })

  return (
    <div>
      <button onClick={shuffle}>Shuffle</button>
      <ul>
        {items.map(item => (
          <li key={item.id} className="item">{item.label}</li>
        ))}
      </ul>
    </div>
  )
}
```

每次 shuffle 时，每项滑到新位置——FLIP 算法保证零卡顿。

## 陷阱

### 在 React 里抓快照的时机

`useEffect` 在 DOM 更新**之后**才跑——抓不到旧状态。`useLayoutEffect` 在更新之后但浏览器绘制之前——可以正确比较前后差异。

正确流程：

1. **事件回调** 里抓 state（DOM 还是旧的）
2. 触发 `setState`
3. React 渲染、commit
4. **`useLayoutEffect`** 里 `Flip.from`（DOM 是新的）

### 元素必须是同一个 DOM 节点

Flip 用元素引用（不是 class 名）追踪——如果 React 重新 mount 节点（key 变了），Flip 找不到对应——动画不会播。

确保 key 稳定，让 React 复用 DOM 节点。

### 与 ScrollTrigger / pin 一起用的 refresh

布局变化后，已有的 ScrollTrigger 位置可能错。Flip 动画完成后手动刷新：

```tsx
Flip.from(state, { onComplete: () => ScrollTrigger.refresh() })
```

### 跨容器移动需要保留同一个元素

```tsx
{inLeft && <div className="item">A</div>}  // ❌ React 卸载、新建
{!inLeft && <div className="item">A</div>}  // 不同节点，Flip 找不到
```

要 portal / 让 React 复用同一节点。Motion 的 `layoutId` 在这点上更省心——它处理了 React 节点变化。

## 小结

- Flip = GSAP 的 FLIP 布局动画——记录前态、改 DOM、播放过渡
- 三步用法：`getState` → DOM 变化 → `from`
- React 集成模式：事件里 `getState` + `useLayoutEffect` 里 `Flip.from`
- `scale: true` 用 scale 实现尺寸变化、`absolute: true` 临时 absolute 防止 layout 干扰
- 跨容器移动 / 列表排序 / 折叠展开都能搞定
- **React 项目里多数场景优先用 Motion 的 `layout` / `layoutId`** ——更省事
- GSAP Flip 适合：已用 GSAP 主导动画 / 需要精细控制 / 与 timeline 协作
