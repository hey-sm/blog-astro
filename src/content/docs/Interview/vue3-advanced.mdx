## 一、keep-alive

### 核心作用

`<keep-alive>` 是 Vue 的内置组件，用于**缓存组件实例**，避免重复销毁和创建。

#### 适用场景

- **Tab 页切换**：保留用户在不同 Tab 的浏览状态
- **表单填写**：切换页面后保留已填写的内容
- **列表滚动位置**：返回列表时保持之前的滚动位置

### 基本用法

```vue
<template>
  <keep-alive>
    <component :is="currentComponent"></component>
  </keep-alive>
</template>

<script setup>
import { ref } from 'vue'
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'

const currentComponent = ref(ComponentA)
</script>
```

### 配置属性

#### include / exclude

指定哪些组件需要缓存或不缓存。

```vue
<!-- 只缓存 name 为 ComponentA 的组件 -->
<keep-alive include="ComponentA">
  <component :is="currentComponent"></component>
</keep-alive>

<!-- 缓存多个组件（逗号分隔） -->
<keep-alive include="ComponentA,ComponentB">
  <component :is="currentComponent"></component>
</keep-alive>

<!-- 使用正则表达式 -->
<keep-alive :include="/Component[AB]/">
  <component :is="currentComponent"></component>
</keep-alive>

<!-- 排除特定组件 -->
<keep-alive exclude="ComponentC">
  <component :is="currentComponent"></component>
</keep-alive>
```

⚠️ **注意**：`include` 和 `exclude` 匹配的是组件的 `name` 选项，而不是文件名。

#### max

限制最大缓存组件数量，超出时使用 **LRU（Least Recently Used）策略**淘汰最久未使用的组件。

```vue
<keep-alive :max="3">
  <component :is="currentComponent"></component>
</keep-alive>
```

### 生命周期钩子

被 `<keep-alive>` 缓存的组件会新增两个生命周期钩子：

- **`onActivated`**：组件被激活（从缓存中恢复）时调用
- **`onDeactivated`**：组件被停用（放入缓存）时调用

```vue
<script setup>
import { onActivated, onDeactivated, onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('组件挂载')
})

onActivated(() => {
  console.log('组件激活（从缓存中恢复）')
  // 可在此刷新数据
})

onDeactivated(() => {
  console.log('组件停用（放入缓存）')
})

onUnmounted(() => {
  console.log('组件卸载（从缓存中移除）')
})
</script>
```

### 结合路由使用

```vue
<template>
  <router-view v-slot="{ Component }">
    <keep-alive :include="['Home', 'About']">
      <component :is="Component" />
    </keep-alive>
  </router-view>
</template>
```

1. **缓存机制**：使用 `Map` 数据结构缓存 VNode 实例
2. **DOM 操作**：组件被缓存时，DOM 会从文档流中 **detached**（分离），激活时重新 **attached**（附加）
3. **LRU 策略**：当缓存数量超过 `max` 时，删除最久未使用的组件

## 二、异步组件

### 定义异步组件

Vue3 使用 `defineAsyncComponent` 函数定义异步组件。

```js
import { defineAsyncComponent } from 'vue'

// 基本用法
const AsyncComponent = defineAsyncComponent(() => 
  import('./components/MyComponent.vue')
)
```

### 配置选项

```js
const AsyncComponent = defineAsyncComponent({
  // 加载函数
  loader: () => import('./components/Heavy.vue'),
  
  // 加载中显示的组件
  loadingComponent: LoadingSpinner,
  
  // 加载失败显示的组件
  errorComponent: ErrorComponent,
  
  // 展示加载组件前的延迟时间（默认 200ms）
  delay: 200,
  
  // 超时时间（超时则显示错误组件）
  timeout: 3000
})
```

### 在组件中使用

```vue
<template>
  <AsyncComponent />
</template>

<script setup>
import { defineAsyncComponent } from 'vue'

const AsyncComponent = defineAsyncComponent(() => 
  import('./components/MyComponent.vue')
)
</script>
```

### Suspense 组件

`<Suspense>` 是 Vue3 的实验性特性，用于统一管理异步组件的加载状态。

```vue
<template>
  <Suspense>
    <!-- 默认插槽：异步内容 -->
    <template #default>
      <AsyncComponent />
    </template>
    
    <!-- 后备插槽：加载中显示的内容 -->
    <template #fallback>
      <LoadingSpinner />
    </template>
  </Suspense>
</template>
```

1. **路由懒加载**：减少初始包体积

```js
const routes = [
  {
    path: '/about',
    component: () => import('./views/About.vue')
  }
]
```

2. **条件渲染组件**：只在需要时加载

```vue
<template>
  <button @click="showModal = true">打开弹窗</button>
  <AsyncModal v-if="showModal" />
</template>
```

3. **大型模块按需加载**：如富文本编辑器、图表库等

## 三、Vue Router

### Vue Router 4.x 核心变化

#### 1. 创建路由实例

Vue Router 4.x 使用 `createRouter` 函数。

```js
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      name: 'Home',
      component: () => import('./views/Home.vue')
    },
    {
      path: '/about',
      name: 'About',
      component: () => import('./views/About.vue')
    }
  ]
})

export default router
```

#### 2. 组合式 API

使用 `useRouter` 和 `useRoute` 替代 `this.$router` 和 `this.$route`。

```vue
<script setup>
import { useRouter, useRoute } from 'vue-router'

const router = useRouter()
const route = useRoute()

// 编程式导航
const goToAbout = () => {
  router.push('/about')
}

// 访问路由参数
console.log(route.params.id)
console.log(route.query.page)
</script>
```

### 路由模式

#### History 模式

使用 HTML5 History API，URL 更美观但需要服务器配置。

```js
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(), // URL: /about
  routes: [...]
})
```

#### Hash 模式

使用 URL 的 hash（#），无需服务器配置。

```js
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(), // URL: /#/about
  routes: [...]
})
```

### 导航守卫

#### 全局守卫

```js
// 全局前置守卫
router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login')
  } else {
    next()
  }
})

// 全局解析守卫
router.beforeResolve((to, from, next) => {
  // 在导航被确认前调用
  next()
})

// 全局后置钩子
router.afterEach((to, from) => {
  // 导航完成后调用（无 next 参数）
  document.title = to.meta.title || 'App'
})
```

#### 路由独享守卫

```js
const routes = [
  {
    path: '/admin',
    component: Admin,
    beforeEnter: (to, from, next) => {
      if (isAdmin()) {
        next()
      } else {
        next('/forbidden')
      }
    }
  }
]
```

## 四、状态管理

### Vuex 核心概念

#### State

集中存储应用状态。

```js
const store = createStore({
  state: {
    count: 0,
    user: null
  }
})

// 访问 state
console.log(store.state.count)
```

#### Getter

派生状态，类似于计算属性。

```js
const store = createStore({
  state: {
    todos: [
      { id: 1, text: 'Learn Vue', done: true },
      { id: 2, text: 'Build App', done: false }
    ]
  },
  getters: {
    doneTodos: (state) => {
      return state.todos.filter(todo => todo.done)
    },
    doneTodosCount: (state, getters) => {
      return getters.doneTodos.length
    }
  }
})

// 访问 getter
console.log(store.getters.doneTodosCount)
```

#### Mutation

同步修改 state 的唯一方式。

```js
const store = createStore({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++
    },
    incrementBy(state, payload) {
      state.count += payload
    }
  }
})

// 提交 mutation
store.commit('increment')
store.commit('incrementBy', 10)
```

#### Action

处理异步操作，通过提交 mutation 修改状态。

```js
const store = createStore({
  state: {
    user: null
  },
  mutations: {
    setUser(state, user) {
      state.user = user
    }
  },
  actions: {
    async fetchUser({ commit }, userId) {
      const user = await api.getUser(userId)
      commit('setUser', user)
    }
  }
})

// 分发 action
store.dispatch('fetchUser', 123)
```

#### Module

模块化管理大型应用的状态。

```js
const moduleA = {
  namespaced: true,
  state: { count: 0 },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment')
      }, 1000)
    }
  }
}

const store = createStore({
  modules: {
    a: moduleA
  }
})

// 访问模块
store.state.a.count
store.commit('a/increment')
store.dispatch('a/incrementAsync')
```

### Pinia（Vue3 推荐）

#### Pinia 优势

- ✅ **无 Mutations**：直接通过 Actions 修改状态
- ✅ **扁平化结构**：无嵌套模块，简化代码
- ✅ **TypeScript 友好**：完整的类型推断
- ✅ **支持多种语法**：Options Store 和 Setup Store

#### 定义 Store（Options Store）

```js
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    name: 'John',
    age: 25
  }),
  getters: {
    doubleAge: (state) => state.age * 2
  },
  actions: {
    async fetchUser(id) {
      const user = await api.getUser(id)
      this.name = user.name
      this.age = user.age
    }
  }
})
```

#### 定义 Store（Setup Store）

```js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useUserStore = defineStore('user', () => {
  const name = ref('John')
  const age = ref(25)
  const doubleAge = computed(() => age.value * 2)
  
  async function fetchUser(id) {
    const user = await api.getUser(id)
    name.value = user.name
    age.value = user.age
  }
  
  return { name, age, doubleAge, fetchUser }
})
```

#### 在组件中使用

```vue
<script setup>
import { useUserStore } from '@/stores/user'
import { storeToRefs } from 'pinia'

const userStore = useUserStore()

// 解构保持响应性
const { name, age, doubleAge } = storeToRefs(userStore)

// 直接修改 state
userStore.name = 'Jane'

// 批量修改
userStore.$patch({
  name: 'Jane',
  age: 30
})

// 调用 action
userStore.fetchUser(123)
</script>
```

## 五、性能优化

### 响应式优化

#### Proxy 替代 Object.defineProperty

Vue3 使用 Proxy 实现响应式，性能更优。

**优势**：
- ✅ 可以监听数组索引和长度变化
- ✅ 可以监听对象属性的新增和删除
- ✅ 支持 Map、Set、WeakMap、WeakSet

#### 惰性依赖追踪

只追踪实际访问的属性，减少开销。

### 编译优化

#### 静态提升（Static Hoisting）

将静态节点提升到渲染函数外部，避免重复创建。

```js
// 优化前
function render() {
  return h('div', [
    h('p', 'Static Text'),
    h('p', dynamicText)
  ])
}

// 优化后
const hoisted = h('p', 'Static Text')
function render() {
  return h('div', [
    hoisted,
    h('p', dynamicText)
  ])
}
```

#### Patch Flag

标记动态节点类型，精准更新。

```js
// 文本节点
<div>{{ text }}</div>  // PatchFlag.TEXT

// 属性
<div :class="cls"></div>  // PatchFlag.CLASS

// 样式
<div :style="style"></div>  // PatchFlag.STYLE
```

#### Block Tree

将动态节点收集到一个数组中，跳过静态节点比对。

### 组件优化

#### v-memo

缓存子树，只在依赖变化时重新渲染。

```vue
<div v-memo="[value1, value2]">
  <!-- 只有 value1 或 value2 变化时才重新渲染 -->
</div>
```

#### shallowRef / shallowReactive

只对第一层属性进行响应式处理。

```js
import { shallowRef } from 'vue'

const state = shallowRef({
  nested: { count: 0 }
})

// 不会触发更新
state.value.nested.count++

// 会触发更新
state.value = { nested: { count: 1 } }
```

#### 异步组件与代码分割

```js
const HeavyComponent = defineAsyncComponent(() =>
  import('./components/Heavy.vue')
)
```

### 资源优化

#### 路由懒加载

```js
const routes = [
  {
    path: '/about',
    component: () => import('./views/About.vue')
  }
]
```

#### Tree Shaking

移除未使用的代码，减少打包体积。

```js
// 按需导入
import { ref, computed } from 'vue'

// ❌ 不要全量导入
import Vue from 'vue'
```

#### 虚拟滚动

处理大量列表数据时，只渲染可见区域。

```vue
<template>
  <virtual-scroller
    :items="items"
    :item-height="50"
  >
    <template #default="{ item }">
      <div>{{ item.name }}</div>
    </template>
  </virtual-scroller>
</template>
```

## 六、Vue2 与 Vue3 的区别

### 响应式系统

| 特性 | Vue2 | Vue3 |
|------|------|------|
| **实现方式** | Object.defineProperty | Proxy |
| **监听数组** | 需重写数组方法 | 原生支持 |
| **新增属性** | 需要 `$set` | 自动响应 |
| **删除属性** | 需要 `$delete` | 自动响应 |
| **性能** | 初始化时递归遍历 | 惰性代理 |

### API 设计

#### Vue2（Options API）

```js
export default {
  data() {
    return { count: 0 }
  },
  computed: {
    double() {
      return this.count * 2
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
```

#### Vue3（Composition API）

```js
import { ref, computed } from 'vue'

export default {
  setup() {
    const count = ref(0)
    const double = computed(() => count.value * 2)
    const increment = () => count.value++
    
    return { count, double, increment }
  }
}
```

## 七、SPA（单页面应用）

### 核心特点

- ✅ **无刷新跳转**：通过前端路由实现页面切换
- ✅ **前后端分离**：前端独立开发和部署

- ❌ **首屏加载慢**：需要加载完整的 JS 包
- ❌ **SEO 不友好**：爬虫难以抓取动态内容

## 八、SSR（服务端渲染）

### 工作原理

1. **服务器端**：Vue 组件渲染为 HTML 字符串
2. **返回 HTML**：包含完整内容的 HTML 发送给浏览器
3. **客户端激活（Hydration）**：Vue 接管静态 HTML，绑定事件和响应式

### SSR 优势

- ✅ **SEO 友好**：爬虫可直接抓取完整内容
- ✅ **首屏快**：用户立即看到内容
- ✅ **更好的性能指标**：FCP、LCP 更优

### SSR 劣势

- ❌ **服务器负载高**：每次请求都要渲染
- ❌ **开发复杂度**：需要考虑服务端/客户端差异
- ❌ **部署成本高**：需要 Node.js 服务器

```js
    '/dashboard/**': { ssr: false },
    
    // 增量静态生成（ISR）
    '/blog/**': { swr: 3600 } // 1小时缓存
  }
})
```

```js
  ttl: 1000 * 60 * 15 // 15分钟

  cache
```
