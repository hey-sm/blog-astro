## 数据类型与内存

### 基本数据类型

JavaScript 有 7 种基本数据类型：

- `String`、`Number`、`Boolean`、`Null`、`Undefined`
- `Symbol`（ES6）
- `BigInt`（ES2020）

**存储特点**：存储在栈内存中，变量直接持有值，赋值时是值的复制。

```js
let a = 10;
let b = a;
b = 20;
console.log(a); // 10 - b 的修改不影响 a
```

### 引用数据类型

主要包括 `Object`（普通对象、`Array`、`Function`、`Date` 等）。

**存储特点**：对象实体存储在堆内存中，变量保存的是堆内存地址的引用。

```js
let obj1 = { name: 'Mickey' };
let obj2 = obj1;
obj2.name = 'Donald';
console.log(obj1.name); // 'Donald' - obj1 和 obj2 指向同一内存地址
```

### 类型判断

**`typeof`**：判断基本类型，但 `typeof null` 返回 `'object'`，无法细分对象类型。

**`instanceof`**：基于原型链判断对象是否为某构造函数的实例。

```js
console.log(typeof 123);           // "number"
console.log(typeof null);          // "object" (历史遗留问题)
console.log([] instanceof Array);  // true
```

## 变量声明：var、let 与 const

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函数作用域 | 块级作用域 | 块级作用域 |
| 变量提升 | 存在（声明提升） | 不存在（TDZ） | 不存在（TDZ） |
| 重复声明 | 允许 | 不允许 | 不允许 |
| 重新赋值 | 允许 | 允许 | 不允许 |

**关键面试点**：

```js
// var 的变量提升
console.log(foo); // undefined
var foo = 'Hello';

// let 的暂时性死区（TDZ）
// console.log(bar); // ReferenceError
let bar = 'World';

// const 的不变性：引用不变，内容可变
const person = { name: 'Mickey', age: 30 };
// person = {}; // TypeError
person.age = 31; // 允许 - 修改对象属性
```

## 运算符

### 核心运算符

- **算术**：`+`、`-`、`*`、`/`、`%`、`**`（幂运算）
- **赋值**：`=`、`+=`、`-=`、`*=`、`/=`
- **比较**：`==`、`===`、`!=`、`!==`、`>`、`<`、`>=`、`<=`
- **逻辑**：`&&`、`||`、`!`

### 短路特性（重要面试点）

```js
// && (与)：第一个为假直接返回，不计算第二个
const result1 = 0 && console.log('不会执行');
console.log(result1); // 0

const user = { name: 'Mickey' };
const userName = user && user.name; // 安全访问嵌套属性

// || (或)：第一个为真直接返回
const result2 = 'Hello' || console.log('不会执行');

// ES6 之前的默认值实现
function greet(name) {
  name = name || 'Guest';
  console.log(`Welcome, ${name}`);
}
```

## 流程控制

### 条件语句

- **`if...else if...else`**：处理多条件判断
- **`switch`**：基于表达式值执行不同操作，比长串 if 更清晰

```js
function getFruitColor(fruit) {
  switch (fruit) {
    case 'apple':
      return 'red';
    case 'banana':
      return 'yellow';
    default:
      return 'unknown';
  }
}
```

### 循环语句

- **`for`**：循环次数已知时使用
- **`while` / `do...while`**：条件循环
- **`for...of`**（ES6）：遍历可迭代对象的值

```js
const numbers = [10, 20, 30];

// 传统 for 循环
for (let i = 0; i < numbers.length; i++) {
  console.log(`索引${i}: ${numbers[i]}`);
}

// for...of 更简洁
for (const number of numbers) {
  console.log(`值: ${number}`);
}
```

## 函数

### 三种定义方式

**函数声明**：存在函数提升

```js
function sum(a, b) {
  return a + b;
}
```

**函数表达式**：遵循变量提升规则

```js
const multiply = function(a, b) {
  return a * b;
};
```

**箭头函数（ES6）**：语法简洁，`this` 由词法作用域决定

```js
const subtract = (a, b) => a - b;
```

### 函数提升面试题

```js
console.log(declaredSum(5, 5)); // 10 - 函数声明被提升

function declaredSum(a, b) {
  return a + b;
}

// console.log(expressedSum(5, 5)); 
// TypeError (var) 或 ReferenceError (let/const)

const expressedSum = function(a, b) {
  return a + b;
};
```

## 对象

### 创建与访问

- 字面量创建：`const obj = {};`
- **点表示法**：`obj.key`（有效标识符）
- **方括号表示法**：`obj['key']`（特殊字符/变量/动态计算）

### 属性操作

```js
const car = {
  make: 'Toyota',
  model: 'Camry',
  'year-of-manufacture': 2022, // 特殊字符键
  start: function() {
    console.log('Engine started!');
  }
};

// 访问
console.log(car.make);                      // 'Toyota'
console.log(car['year-of-manufacture']);    // 2022 - 必须用方括号

// 修改/添加/删除
car.model = 'Corolla';
car.color = 'blue';
delete car.color;

// 遍历 - 推荐使用 hasOwnProperty
for (const key in car) {
  if (Object.prototype.hasOwnProperty.call(car, key)) {
    console.log(`${key}: ${car[key]}`);
  }
}
```

## 数组

### 常用方法分类

**改变原数组**：
- `push()` / `pop()`：末尾添加/删除
- `unshift()` / `shift()`：开头添加/删除
- `splice()`：任意位置添加/删除/替换
- `sort()`：排序

**不改变原数组**：
- `concat()`：合并数组
- `slice()`：提取部分
- `forEach()`：遍历
- `map()`：转换生成新数组
- `filter()`：过滤生成新数组
- `find()`：查找第一个满足条件的元素

### map vs forEach（重要对比）

```js
const ids = [1, 2, 3, 4];

// forEach：仅迭代，无返回值
ids.forEach(id => {
  console.log(`Processing ID: ${id}`);
});

// map：转换数组，返回新数组
const urls = ids.map(id => `/users/${id}`);
console.log(urls); // ['/users/1', '/users/2', '/users/3', '/users/4']
console.log(ids);  // [1, 2, 3, 4] - 原数组不变
```

## DOM 操作与事件

### 高效 DOM 节点操作

使用 `DocumentFragment` 避免多次重排重绘：

```js
const list = document.getElementById('my-list');
const fragment = document.createDocumentFragment();

for (let i = 0; i < 1000; i++) {
  const item = document.createElement('li');
  item.textContent = `Item ${i + 1}`;
  fragment.appendChild(item); // 先附加到 fragment
}

list.appendChild(fragment); // 一次性添加，只触发一次重排
```

### 事件委托

利用事件冒泡，在父元素统一处理子元素事件：

```js
const parentList = document.getElementById('parent-list');

parentList.addEventListener('click', function(event) {
  // event.target 是实际被点击的元素
  if (event.target && event.target.nodeName === 'LI') {
    console.log(`点击了: ${event.target.textContent}`);
  }
});
```

## AJAX 与本地存储

### AJAX - fetch API

基于 Promise 的现代异步请求方案：

```js
async function fetchUserData() {
  try {
    const response = await fetch('https://api.example.com/user/1');
    
    // fetch 不会因 HTTP 错误状态码 reject，需手动检查
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Fetch failed:', error);
  }
}
```

### 本地存储方案对比

| 方案 | 生命周期 | 存储大小 | 与服务器通信 |
|------|----------|----------|--------------|
| localStorage | 永久（除非手动清除） | 约 5MB | 否（仅客户端） |
| sessionStorage | 标签页关闭清除 | 约 5MB | 否（仅客户端） |
| Cookie | 可设置过期时间 | 约 4KB | 每次 HTTP 请求携带 |

## 闭包

闭包是函数和其词法作用域的组合，即函数能够访问并记住其外部作用域的变量，即使外部函数已经执行完毕。

**形成条件**：
- 函数嵌套
- 内部函数引用外部函数的变量
- 外部函数返回内部函数（或以其他方式使内部函数被外部访问）

**典型应用**：
- 数据私有化：创建私有变量，外部无法直接访问
- 模块化封装：实现模块模式，暴露公共接口
- 函数工厂：动态创建带有特定配置的函数
- 回调函数：事件监听、异步操作中保留上下文

**注意事项**：
- 闭包会保持对外部变量的引用，可能造成内存泄漏
- 循环中创建闭包要注意变量共享问题（用 let 或 IIFE 解决）

#### 防抖与节流

| 对比项 | 防抖 | 节流 |
|--------|------|------|
| 定义 | 延迟执行，停止触发后等待 N 毫秒才执行 | 控制频率，指定时间内只执行一次 |
| 核心逻辑 | 触发时启动定时器，再次触发则清除旧定时器重新计时 | 设定时间间隔，检查是否到达间隔，未到达则忽略 |
| 执行时机 | 停止触发后延迟执行 | 按固定间隔执行 |
| 执行次数 | 可能只执行一次（最后一次） | 持续触发会多次执行 |
| 适用场景 | 搜索框输入、窗口 resize、表单验证 | 滚动加载、按钮防重、鼠标移动 |
| 形象比喻 | 电梯等人齐了再走 | 公交车到点就发车 |

### 浏览器渲染

浏览器渲染是将 HTML、CSS、JS 转换为屏幕像素的过程，核心是构建 DOM 树、CSSOM 树、渲染树，然后布局和绘制。

**渲染流程**：
1. **解析 HTML** → 构建 DOM 树
2. **解析 CSS** → 构建 CSSOM 树
3. **合并** → DOM + CSSOM = 渲染树（Render Tree）
4. **布局（Layout/Reflow）** → 计算元素位置和尺寸
5. **绘制（Paint）** → 绘制像素到屏幕
6. **合成（Composite）** → 将图层合并显示

**关键概念**：
- **回流（Reflow）**：元素几何属性变化（位置、尺寸），需重新计算布局，成本高
- **重绘（Repaint）**：元素样式变化（颜色、背景），不影响布局，成本较低
- **合成（Composite）**：只改变图层合成，不触发布局和绘制，性能最优（如 transform、opacity）

**性能优化**：
- 减少回流：批量修改 DOM、使用 DocumentFragment、避免逐条修改样式
- 使用 transform/opacity 替代 top/left：触发合成而非回流
- CSS 放 `<head>`、JS 放 `</body>` 前：避免阻塞渲染
- 使用 `will-change` 或 `transform: translateZ(0)` 提升为独立图层

**阻塞渲染**：
- CSS 阻塞渲染：浏览器等 CSSOM 构建完才绘制
- JS 阻塞解析：`<script>` 会暂停 HTML 解析，用 `async` / `defer` 异步加载

### 事件循环

事件循环是 JS 单线程下管理异步任务的核心机制，核心是宏任务、微任务、渲染时机三者的执行顺序。

**执行顺序**：
1. 同步代码
2. 微任务（Promise、async/await）
3. UI 渲染
4. 宏任务（setTimeout、网络请求）
5. 循环往复

**关键规则**：
- 微任务优先级高于宏任务
- 微任务会阻塞渲染，必须全部执行完才会更新页面
- 宏任务一定在渲染之后执行
