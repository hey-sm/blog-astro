# SSR/SSG

Vue 的服务端渲染和静态站点生成方案。

## Nuxt 3

### 创建项目

```bash
npx nuxi@latest init my-app
cd my-app
npm install
npm run dev
```

### 目录结构

```text
my-app/
├── pages/           # 自动路由
│   ├── index.vue
│   └── about.vue
├── components/      # 自动导入
├── composables/     # 组合式函数
├── layouts/         # 布局
├── server/          # 服务端 API
├── app.vue          # 根组件
└── nuxt.config.ts   # 配置
```

### 页面组件

```vue filename="pages/index.vue"
<script setup lang="ts">
const { data } = await useFetch('/api/users')
</script>

<template>
  <div>
    <h1>Home</h1>
    <ul>
      <li v-for="user in data" :key="user.id">
        {{ user.name }}
      </li>
    </ul>
  </div>
</template>
```

### 服务端 API

```ts filename="server/api/users.ts"
export default defineEventHandler(() => {
  return [
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
  ]
})
```

## VitePress

### 创建项目

```bash
npm install -D vitepress
npx vitepress init
```

### 配置

```ts filename=".vitepress/config.ts"
import { defineConfig } from 'vitepress'

export default defineConfig({
  title: 'My Docs',
  description: 'Documentation site',
  themeConfig: {
    nav: [
      { text: 'Home', link: '/' },
      { text: 'Guide', link: '/guide/' }
    ],
    sidebar: [
      {
        text: 'Guide',
        items: [
          { text: 'Introduction', link: '/guide/introduction' },
          { text: 'Getting Started', link: '/guide/getting-started' }
        ]
      }
    ]
  }
})
```

### Markdown 文件

```md filename="guide/introduction.md"
# Introduction

This is the introduction page.

<script setup>
import { ref } from 'vue'

const count = ref(0)
</script>

<button @click="count++">{{ count }}</button>
```
