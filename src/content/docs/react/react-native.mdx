# React Native

React Native（RN）是 Meta 开源的**跨平台移动应用框架**，让开发者用 React 的心智模型构建 iOS / Android 原生应用。

> "用 React 写**真实原生组件**——不是 Web View。"

本章覆盖：

- RN 与 React (Web) 的关键差异
- Expo 与裸 RN 的选择
- 核心组件与样式系统
- 导航、状态、网络
- 平台差异处理与性能注意

## 心智模型

### RN 不是 Web

RN 不渲染 DOM。`<View>` 编译为 iOS 的 `UIView` 与 Android 的 `android.view.View`；`<Text>` 编译为对应原生文本组件。

```tsx
import { View, Text, Pressable } from 'react-native'

export default function App() {
  return (
    <View style={{ padding: 20 }}>
      <Text style={{ fontSize: 18 }}>Hello, RN!</Text>
      <Pressable onPress={() => alert('clicked')}>
        <Text>Tap me</Text>
      </Pressable>
    </View>
  )
}
```

| Web React | React Native |
| --------- | ------------ |
| `<div>` | `<View>` |
| `<span>` / `<p>` | `<Text>`（**所有文字必须包在 `<Text>` 里**） |
| `<img>` | `<Image>` |
| `<input>` | `<TextInput>` |
| `<button>` | `<Pressable>` / `<Button>` |
| `<a>` | `<Pressable>` + `Linking.openURL` |
| `<ul><li>` | `<FlatList>` / `<SectionList>` |
| `<form>` | 无对应；自己拼 `TextInput + Button` |
| CSS | StyleSheet（JS 对象） |
| className | style |

### 渲染流水线（New Architecture）

RN 0.76+ 默认启用 **新架构**（Fabric + TurboModules）：

```text
JS thread (React)        ──┐
                           │  JSI（同步桥接，无序列化）
Native thread (Fabric)   ──┘
  ├─ Fabric Renderer：把 React 树转换为原生 Shadow Tree
  └─ TurboModules：原生模块按需加载，类型化接口
```

旧架构通过异步 JSON 桥（Bridge）通信，是众多性能问题的根源；新架构去除桥接，性能与启动时间显著改善。

## Expo

[Expo](https://expo.dev) 是 RN 的官方推荐工具链：

- **零 Xcode / Android Studio 启动**：扫码即运行
- **Expo Router**：文件系统路由
- **EAS Build**：云端构建签名 / 发布
- **OTA 更新**：用户无需重新装包即可获取 JS 更新
- **大量包装好的原生模块**：相机、位置、通知、生物识别等

> Meta、Expo、React Native 团队联合推荐：**新项目从 Expo 开始**。除非项目对原生侧有重度定制需求。

```bash
pnpm create expo@latest my-app
cd my-app
pnpm start
```

启动开发服务器后：

- iOS / Android 真机：扫描二维码（需安装 Expo Go App）
- 模拟器：在终端按 `i`（iOS）或 `a`（Android）

### Expo Router（文件路由）

类似 Next.js App Router 的体验：

```text
app/
├── _layout.tsx              # 根布局
├── index.tsx                # /
├── (tabs)/                  # tab 分组
│   ├── _layout.tsx
│   ├── home.tsx
│   └── settings.tsx
└── posts/
    └── [id].tsx             # /posts/123
```

```tsx filename="app/_layout.tsx"
import { Stack } from 'expo-router'

export default function RootLayout() {
  return <Stack />
}
```

```tsx filename="app/posts/[id].tsx"
import { useLocalSearchParams, Link } from 'expo-router'
import { View, Text } from 'react-native'

export default function Post() {
  const { id } = useLocalSearchParams<{ id: string }>()
  return (
    <View>
      <Text>Post {id}</Text>
      <Link href="/">Home</Link>
    </View>
  )
}
```

## 样式：StyleSheet

RN 不支持 CSS 文件，所有样式以 JS 对象表达：

```tsx
import { StyleSheet, View, Text } from 'react-native'

function Card() {
  return (
    <View style={styles.card}>
      <Text style={styles.title}>Title</Text>
    </View>
  )
}

const styles = StyleSheet.create({
  card: {
    padding: 16,
    backgroundColor: 'white',
    borderRadius: 8,
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 4,
    elevation: 2                              // Android 阴影
  },
  title: { fontSize: 18, fontWeight: '600' }
})
```

`StyleSheet.create` 与裸对象在功能上等价，但提供：

- 静态分析（IDE 能识别 key）
- 性能优化（哈希引用代替每次新建）

### 布局：Flexbox

RN 默认 `flex-direction: column`（与 Web 默认 `row` 不同）。除此之外 Flexbox 子集与 Web 一致：

```tsx
<View style={{ flex: 1, flexDirection: 'row', justifyContent: 'space-between' }}>
  <Text>Left</Text>
  <Text>Right</Text>
</View>
```

不支持：`grid`、`float`、复杂选择器、伪类。

### NativeWind（Tailwind for RN）

```bash
pnpm add nativewind
```

```tsx
import { View, Text } from 'react-native'

<View className="p-4 bg-white rounded-lg shadow">
  <Text className="text-lg font-semibold">Hello</Text>
</View>
```

NativeWind 把 Tailwind 类名编译为 RN style 对象。Web 与 RN 团队共享的样式约定是其最大优势——一份心智，两端复用。

### 其他主流方案

- **Tamagui**：跨 Web + RN 的统一组件库，编译期优化
- **styled-components / Emotion**：可用于 RN，运行时开销
- **Restyle (Shopify)**：类型化主题 + 工具函数
- **Gluestack**：基于 Tailwind 的组件库

## 列表渲染：FlatList

```tsx
import { FlatList, View, Text } from 'react-native'

<FlatList
  data={items}
  keyExtractor={item => item.id}
  renderItem={({ item }) => (
    <View><Text>{item.name}</Text></View>
  )}
  ItemSeparatorComponent={() => <View style={{ height: 1, backgroundColor: '#eee' }} />}
  ListEmptyComponent={() => <Text>No items</Text>}
  ListHeaderComponent={Header}
  refreshControl={<RefreshControl refreshing={refreshing} onRefresh={refetch} />}
  onEndReached={loadMore}
  onEndReachedThreshold={0.5}
/>
```

`FlatList` 提供虚拟滚动——仅渲染屏幕可见区域附近的项，是大列表的标准选择。

性能加分项：

- `keyExtractor` 必须返回稳定 ID
- `renderItem` 用 `useCallback` / 提取为独立组件
- `getItemLayout` 提供等高项的尺寸，让 `FlatList` 跳过测量
- `removeClippedSubviews={true}`：进一步减少内存
- 复杂场景使用 [`@shopify/flash-list`](https://github.com/Shopify/flash-list)：FlatList 的高性能替代

## 网络与数据

```tsx
const res = await fetch('https://api.example.com/users')
const users = await res.json()
```

`fetch` API 与 Web 一致。TanStack Query / SWR / Axios 在 RN 中无差别使用——它们的实现不依赖 Web 平台 API。

```tsx
import { useQuery } from '@tanstack/react-query'

function UserList() {
  const { data, isPending } = useQuery({
    queryKey: ['users'],
    queryFn: () => fetch('/api/users').then(r => r.json())
  })
  // ...
}
```

详见 [数据请求](/react/api-calls)。

## 平台差异

```tsx
import { Platform } from 'react-native'

const styles = {
  paddingTop: Platform.OS === 'ios' ? 20 : 0
}

if (Platform.OS === 'web') {
  // expo-router 也支持 Web
}
```

平台特定文件名：

```text
Button.ios.tsx
Button.android.tsx
Button.tsx                    # 兜底
```

构建时按平台自动选择。

## 输入处理

```tsx
import { TextInput, KeyboardAvoidingView, Platform } from 'react-native'

<KeyboardAvoidingView
  behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
  style={{ flex: 1 }}
>
  <TextInput
    value={text}
    onChangeText={setText}                    // RN 是 onChangeText (string)，不是 onChange (event)
    placeholder="Type here"
    keyboardType="email-address"
    autoCapitalize="none"
    secureTextEntry={false}
  />
</KeyboardAvoidingView>
```

注意 RN 的事件签名与 Web 不同——直接接收值，不是事件对象。

## 资产管理

### 图片

```tsx
import { Image } from 'react-native'

// 静态资产
<Image source={require('./logo.png')} />

// 网络
<Image source={{ uri: 'https://example.com/logo.png' }} style={{ width: 100, height: 100 }} />
```

网络图片**必须显式指定尺寸**，否则不会显示。

#### 高性能图片：expo-image

```tsx
import { Image } from 'expo-image'

<Image
  source={uri}
  contentFit="cover"
  transition={200}
  cachePolicy="memory-disk"
/>
```

支持渐进加载、缓存、动画格式（WebP、AVIF）、占位符。

### 字体

```tsx

export default function App() {

    'Inter': require('./assets/Inter-Regular.ttf'),
    'Inter-Bold': require('./assets/Inter-Bold.ttf')
  })
  if (!loaded) return null
  // ...
}
```

## 性能要点

1. **避免在主线程跑 CPU 密集任务**：使用 [`react-native-worklets-core`](https://github.com/margelo/react-native-worklets-core) 在 UI 线程跑同步逻辑
2. **动画用 Reanimated**：[`react-native-reanimated`](https://docs.swmansion.com/react-native-reanimated/)，动画在原生线程执行
3. **手势用 Gesture Handler**：[`react-native-gesture-handler`](https://docs.swmansion.com/react-native-gesture-handler/)，规避 JS 桥
4. **列表用 FlashList**：替代 FlatList
5. **图像用 expo-image**：替代原生 Image
6. **避免大量 inline style**：每次渲染产生新对象，触发 diff
7. **InteractionManager**：在交互结束后调度任务，避免动画卡顿

```tsx
import { InteractionManager } from 'react-native'

useEffect(() => {
  const handle = InteractionManager.runAfterInteractions(() => {
    fetchData()                               // 等过场动画结束再请求
  })
  return () => handle.cancel()
}, [])
```

## RN 与 Web 跨端

如果同一业务逻辑要同时支持 Web + iOS + Android：

| 方案 | 思路 |
| ---- | ---- |
| **React Native Web** | RN 组件编译为 DOM 元素 |
| **Expo Router 通用应用** | 文件路由跨平台一致，UI 部分按平台差异化 |
| **Tamagui** | 同一个 API，按平台输出最优代码 |
| **共享业务、各自 UI** | 共用 hooks / API / store，UI 各写一份 |

Vercel、Linear、Cal.com 等团队走的是"共享逻辑，各自 UI"——纯逻辑库（Zustand、TanStack Query 等）跨端无缝，UI 层按平台特性各自精雕。

## RN 与 Flutter / Native 选型

| 维度 | React Native | Flutter | 原生 |
| ---- | ------------ | ------- | ---- |
| 语言 | JS / TS | Dart | Swift / Kotlin |
| UI 一致性 | 平台原生组件 | 自绘（一致） | 平台原生 |
| 上手成本 | 已会 React 即可 | 学 Dart + 框架 | 高 |
| 性能 | 接近原生（新架构） | 接近原生 | 最优 |
| 包大小 | 大 | 中 | 小 |
| 生态 | 极广 | 中 | 平台原生 |
| 适合 | Web 团队跨端 | 设计要求绝对统一 | 复杂原生交互 |

如果团队已重度投入 React 生态，RN 是显而易见的选择；如果对**视觉绝对统一**有要求，Flutter 更合适；性能或硬件级深度集成则原生开发不可替代。

## 小结

- React Native 让 React 心智在 iOS / Android 复用，新架构（Fabric + TurboModules）大幅改善性能；
- 新项目从 **Expo + Expo Router** 起步——除非有重度原生定制需求；
- Web 与 RN 共享业务逻辑（Hook、状态、数据请求）极易，UI 层差异通过 Tamagui / NativeWind / 各自实现处理；
- 列表用 FlashList、图像用 expo-image、动画用 Reanimated、手势用 Gesture Handler——"四件套"是性能关键；
- 选型时先看团队熟悉度与设计一致性要求，再决定 RN / Flutter / 原生。
