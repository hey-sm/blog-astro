# 变量

JavaScript 中的变量是**对值的命名引用**。理解变量必须同时理解它的三个维度：**声明方式（`var` / `let` / `const`）**、**作用域**、**生命周期（提升与 TDZ）**。三者相互交织，是面试与日常调试中错误的高发区。

## 声明方式

| 关键字 | 作用域 | 提升 | 重复声明 | 重新赋值 | 全局对象属性 |
| ------ | ------ | ---- | -------- | -------- | ------------ |
| `var` | 函数作用域 | ✅ 提升并初始化为 `undefined` | ✅ | ✅ | ✅（浏览器中挂到 `window`） |
| `let` | 块作用域 | ✅ 提升但不初始化（TDZ） | ❌ | ✅ | ❌ |
| `const` | 块作用域 | ✅ 提升但不初始化（TDZ） | ❌ | ❌（绑定不可变） | ❌ |

### `var`

```js
function foo() {
  if (true) {
    var x = 1
  }
  console.log(x)                              // → 1
}
foo()
```

`var` 声明的变量绑定到**最近的函数作用域或全局作用域**，不受 `if` / `for` 等代码块约束。

### `let`

```js
function foo() {
  if (true) {
    let x = 1
  }
  console.log(x)                              // ✗ ReferenceError: x is not defined
}
```

`let` 引入了**块作用域（block scope）**，由 `{ }` 限定。这与 C / Java 等多数主流语言一致。

### `const`

```js
const PI = 3.14159
PI = 3                                        // ✗ TypeError: Assignment to constant variable.

const obj = { name: 'js' }
obj.name = 'ts'                               // ✓ 合法：仅引用不可变，对象内部仍可修改
console.log(obj)                              // → { name: 'ts' }
```

`const` 锁定的是**变量到值的绑定**，而非值本身的不可变性。若需要"深度不可变"，需要 `Object.freeze` 或采用 immutable 数据结构库。

```js
const obj = Object.freeze({ name: 'js' })
obj.name = 'ts'                               // 严格模式抛错；非严格模式静默失败
console.log(obj.name)                         // → 'js'
```

注意 `Object.freeze` 是**浅冻结**：

```js
const o = Object.freeze({ inner: { v: 1 } })
o.inner.v = 99                                // ✓ 嵌套对象未被冻结
console.log(o.inner.v)                        // → 99
```

## 变量提升（Hoisting）

JavaScript 的执行可拆分为两阶段：**编译阶段（创建词法环境）** 与 **执行阶段（按语句执行）**。所有声明在编译阶段已被识别，而赋值发生在执行阶段——这种现象被习惯性地描述为"提升"。

### `var` 与函数声明的提升

```js
console.log(a)                                // → undefined（已声明，未赋值）
console.log(foo)                              // → [Function: foo]
console.log(bar)                              // → undefined（变量提升，函数赋值未发生）

var a = 1
function foo() {}
var bar = function () {}
```

等价于：

```js
var a                                         // 声明被提升
var bar                                       // 声明被提升
function foo() {}                             // 整个函数声明被提升

console.log(a)                                // undefined
console.log(foo)                              // [Function: foo]
console.log(bar)                              // undefined

a = 1
bar = function () {}
```

**函数声明** vs **函数表达式** 的差异在此显现：函数声明会被整体提升（包括函数体），函数表达式只会提升变量名。

### `let` / `const` 与暂时性死区（TDZ）

`let` 与 `const` 也会被提升，但在词法环境中处于**未初始化（uninitialized）**状态；进入声明语句之前的访问会抛 `ReferenceError`，这段区间称为 **Temporal Dead Zone**：

```js
{
  console.log(x)                              // ✗ ReferenceError: Cannot access 'x' before initialization
  let x = 1
}
```

TDZ 的存在使 `let` / `const` 的语义比 `var` 更接近其他语言的"先声明后使用"。它由规范 [§14.3.2 Let and Const Declarations](https://tc39.es/ecma262/#sec-let-and-const-declarations) 强制要求。

<details>
<summary>TDZ 的实现层面：CreateImmutableBinding / CreateMutableBinding</summary>

进入块作用域时，引擎为 `let x` 调用 `CreateMutableBinding(N, true)`，此时绑定状态为 `uninitialized`；执行到 `let x = 1` 时调用 `InitializeBinding(N, V)`。在两步之间访问该绑定会触发 `ReferenceError`。

参考：[ECMA-262 §9.1.1.1.4 GetBindingValue](https://tc39.es/ecma262/#sec-declarative-environment-records-getbindingvalue-n-s)。
</details>

### 函数提升与变量提升的优先级

```js
console.log(typeof foo)                       // → 'function'

var foo = 1
function foo() {}

console.log(typeof foo)                       // → 'number'
```

函数声明优先级高于 `var` 声明（同名时函数胜出），但执行阶段的赋值会覆盖。

## 命名规则

ECMAScript 标识符遵循以下约定：

1. 首字符必须为 Unicode 字母、`$` 或 `_`；后续字符可包含数字
2. 不能与保留字相同：`break` / `case` / `class` / `const` / `continue` / `debugger` / `default` / `delete` / `do` / `else` / `enum` / `export` / `extends` / `false` / `finally` / `for` / `function` / `if` / `import` / `in` / `instanceof` / `new` / `null` / `return` / `super` / `switch` / `this` / `throw` / `true` / `try` / `typeof` / `var` / `void` / `while` / `with` / `yield`
3. 严格模式额外保留：`implements` / `interface` / `let` / `package` / `private` / `protected` / `public` / `static` / `arguments` / `eval`

```js
let $foo = 1                                  // ✓
let _bar = 2                                  // ✓
let π = 3.14                                  // ✓ Unicode 合法
let 中文变量 = 'ok'                            // ✓ 但不推荐
let 1abc = 1                                  // ✗ SyntaxError
let class = 1                                 // ✗ SyntaxError
```

### 常用约定

- 普通变量、函数：`camelCase`，如 `userName`、`fetchUser`
- 类、构造函数：`PascalCase`，如 `UserService`
- 常量（运行期不变的字面量）：`SCREAMING_SNAKE_CASE`，如 `MAX_RETRIES`
- 私有约定：`_` 前缀（仅约定）；`#` 前缀（语言级私有，自 ES2022）

## 作用域

### 全局作用域

非模块脚本中，顶层 `var` 与函数声明会成为全局对象（浏览器中是 `window`）的属性；`let` / `const` / `class` 不会：

```js
var a = 1
let b = 2
console.log(window.a)                         // → 1
console.log(window.b)                         // → undefined
```

### 函数作用域

每次函数调用都会创建一个新的**执行上下文（Execution Context）**，其中包含独立的词法环境：

```js
function counter() {
  let count = 0
  return () => ++count
}

const c1 = counter()
const c2 = counter()
console.log(c1())                             // → 1
console.log(c1())                             // → 2
console.log(c2())                             // → 1   c1 与 c2 各自捕获独立作用域
```

### 块作用域

`if` / `for` / `while` / `try` 等语法块都构成独立作用域（仅对 `let` / `const` / `class` / `function` 在严格模式下生效）：

```js
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0)
}
// → 0
// → 1
// → 2
```

如果改用 `var`，三个回调会共享同一个 `i`，最终都打印 `3`。这是 `let` 在循环中的著名优势。

### 词法作用域与作用域链

JavaScript 采用**词法作用域**：变量的可见性由**代码书写位置**决定，而非调用位置：

```js
const x = 'outer'

function inner() {
  console.log(x)
}

function outer() {
  const x = 'inner'
  inner()                                     // 即使在 outer 中调用，inner 也仍看到 'outer'
}

outer()                                       // → outer
```

引擎查找变量时，从当前词法环境开始，沿 `[[OuterEnv]]` 链路向外查找，直到全局环境；找不到则抛 `ReferenceError`（写入 `var` 例外，会创建全局变量，不推荐）。

## 实战陷阱

### 循环中的闭包

```js
// ✗ 经典错误：所有定时器打印 3
var timers = []
for (var i = 0; i < 3; i++) {
  timers.push(() => console.log(i))
}
timers.forEach(fn => fn())                    // → 3, 3, 3

// ✓ 用 let 修复：每次迭代产生新绑定
const fixed = []
for (let i = 0; i < 3; i++) {
  fixed.push(() => console.log(i))
}
fixed.forEach(fn => fn())                     // → 0, 1, 2
```

### `const` 不等于"不可变"

```js
const arr = [1, 2, 3]
arr.push(4)                                   // ✓ 仅引用不可变
arr = []                                      // ✗ TypeError
```

### TDZ 误用

```js
function f() {
  console.log(x)                              // ✗ ReferenceError
  let x
}
```

不要假设"变量稍后定义就行"。`let` / `const` 的访问必须严格在声明之后。

## 选择建议

- **默认使用 `const`**：除非确认变量需要重新赋值；
- **在确实需要重新赋值时使用 `let`**：例如循环计数器、可变累加器；
- **避免使用 `var`**：除非维护遗留代码或显式利用其函数作用域语义。
