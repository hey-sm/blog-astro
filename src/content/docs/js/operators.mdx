# 表达式与运算符

JavaScript 中的运算符按**操作数个数**可分为一元（unary）、二元（binary）、三元（ternary）；按**功能**可分为算术、比较、逻辑、位、字符串、赋值、条件、逗号、`typeof`、`delete`、`in`、`instanceof`、`new`、`void`、`spread/rest`、`optional chaining`、`nullish coalescing` 等。本章按功能维度系统梳理。

## 算术运算符

```js
1 + 2                                         // → 3
5 - 3                                         // → 2
2 * 3                                         // → 6
7 / 2                                         // → 3.5
7 % 2                                         // → 1     取余
2 ** 10                                       // → 1024  幂运算（ES2016）

let x = 5
++x                                           // → 6   前置自增：先自增后返回
x++                                           // → 6   后置自增：先返回后自增
console.log(x)                                // → 7

// 一元加减：触发 ToNumber
+'42'                                         // → 42
+''                                           // → 0
+'abc'                                        // → NaN
-true                                         // → -1
```

### 浮点精度

所有 `Number` 都是 IEEE 754 双精度浮点：

```js
0.1 + 0.2                                     // → 0.30000000000000004
0.1 + 0.2 === 0.3                             // → false

// 工程实践：以 EPSILON 为阈值
const equal = (a, b) => Math.abs(a - b) < Number.EPSILON
equal(0.1 + 0.2, 0.3)                         // → true

// 货币计算：用整数（最小单位）或 BigInt
const cents = 19_99 + 30_00                   // → 4999
```

### `%` 与负数

```js
-7 % 3                                        // → -1   结果符号与左操作数一致
7 % -3                                        // → 1

// 数学上"非负的模"，需手工调整
const mod = (a, b) => ((a % b) + b) % b
mod(-7, 3)                                    // → 2
```

## 比较运算符

| 运算符 | 语义 |
| ------ | ---- |
| `==` / `!=` | 宽松相等（详见 [相等性比较](/js/equality)） |
| `===` / `!==` | 严格相等 |
| `<` / `<=` / `>` / `>=` | 关系比较 |
| `in` | 属性是否存在于对象（含原型链） |
| `instanceof` | 对象是否在某构造函数的原型链上 |

```js
'apple' < 'banana'                            // → true     字典序
[1, 2] < [1, 3]                               // → true     '1,2' < '1,3'

'name' in { name: 'js' }                      // → true
'toString' in {}                              // → true     原型链上的属性也算

;[] instanceof Array                          // → true
;[] instanceof Object                         // → true
```

## 逻辑运算符

| 运算符 | 短路条件 | 返回值 |
| ------ | -------- | ------ |
| `&&` | 左侧为 falsy 时 | 第一个 falsy 或最后一个值 |
| `\|\|` | 左侧为 truthy 时 | 第一个 truthy 或最后一个值 |
| `??` | 左侧为 `null` / `undefined` 时不短路 | 左侧非 nullish 则取左，否则取右 |
| `!` | — | 转换为对应布尔值 |

```js
// && 取第一个 falsy 或最后一个值
1 && 2 && 3                                   // → 3
0 && 'never'                                  // → 0

// || 取第一个 truthy 或最后一个值
0 || '' || 'fallback'                         // → 'fallback'
'a' || 'b'                                    // → 'a'

// ?? 仅 null/undefined 时短路
0 ?? 'fallback'                               // → 0
'' ?? 'fallback'                              // → ''
null ?? 'fallback'                            // → 'fallback'

// 不能直接混用 && / ||  与 ??（语法错误，必须加括号）
true || (false ?? 'x')                        // ✓
true || false ?? 'x'                          // ✗ SyntaxError
```

### `||` vs `??`

```js
// ✗ 错误：当 count 合法地为 0 时，被覆盖为默认值 10
function paginate(count) {
  const size = count || 10
  return size
}
paginate(0)                                   // → 10  ← 不期望

// ✓ 正确：仅在 count 是 null/undefined 时取默认
function paginate(count) {
  const size = count ?? 10
  return size
}
paginate(0)                                   // → 0
```

### 逻辑赋值（ES2021）

```js
let a = null
a ||= 'default'                               // a = a || 'default'
a ??= 'fallback'                              // a = a ?? 'fallback'
a &&= 'newvalue'                              // a = a && 'newvalue'
```

## 位运算符

操作数被先 `ToInt32` 截断为 32 位有符号整数。

```js
5 & 3                                         // → 1   AND
5 | 3                                         // → 7   OR
5 ^ 3                                         // → 6   XOR
~5                                            // → -6  按位取反（~n === -(n + 1)）

5 << 1                                        // → 10  左移 1 位 = ×2
5 >> 1                                        // → 2   有符号右移
-1 >> 0                                       // → -1
-1 >>> 0                                      // → 4294967295   无符号右移
```

#### 实用技巧

```js
// 整数化（仅对 32 位整数范围内的数有效，不推荐用于真正的取整）
3.7 | 0                                       // → 3
-3.7 | 0                                      // → -3   注意是向 0 取整

// 判断奇偶
n & 1                                         // 1 → 奇，0 → 偶

// 交换两数（不使用临时变量）
let a = 5, b = 3
a ^= b; b ^= a; a ^= b
console.log(a, b)                             // → 3, 5
```

### 解构赋值

```js
// 数组解构
const [a, b, ...rest] = [1, 2, 3, 4]          // a=1, b=2, rest=[3,4]
;[x, y] = [y, x]                              // 交换变量

// 对象解构
const { name, age = 18, ...others } = { name: 'js', city: 'sh' }
// name='js', age=18, others={ city: 'sh' }

// 重命名
const { name: userName } = { name: 'js' }     // userName='js'

// 嵌套解构
const { a: { b: { c } } } = { a: { b: { c: 1 } } }

// 默认值仅在 undefined 时生效
const [v = 1] = [null]
console.log(v)                                // → null
```

## 字符串运算符

`+` 在任一侧为字符串时执行拼接：

```js
'a' + 'b'                                     // → 'ab'
'1' + 1                                       // → '11'
1 + '1'                                       // → '11'
1 + 2 + '3'                                   // → '33'   先算 1+2，再拼
'1' + 2 + 3                                   // → '123'  从左到右逐步拼

// 模板字符串
const name = 'js'
`Hello, ${name}`                              // → 'Hello, js'

// 标签模板
function tag(strings, ...values) {
  return strings.reduce((acc, s, i) => acc + s + (values[i] ?? ''), '')
}
tag`Hello, ${name}!`                          // → 'Hello, js!'
```

## 条件（三元）运算符

```js
const tag = age >= 18 ? 'adult' : 'minor'

// 嵌套（建议改写为 if-else 或映射表）
const level = score >= 90 ? 'A'
            : score >= 80 ? 'B'
            : score >= 60 ? 'C'
            : 'F'
```

## 逗号运算符

```js
const x = (1, 2, 3)                           // x = 3   返回最后一个表达式

for (let i = 0, j = 10; i < j; i++, j--) {
  // 唯一仍常见的用法：for 循环初始化与递增多个变量
}
```

## `typeof` / `delete` / `void` / `in` / `instanceof`

```js
// typeof：类型字符串
typeof undeclared                             // → 'undefined'   未声明也不抛错

// delete：移除属性，返回布尔
const o = { a: 1 }
delete o.a                                    // → true
'a' in o                                      // → false

// void：求值并返回 undefined（书签 javascript: 链接、避免污染表达式语义）
void 0                                        // → undefined

// in：含原型链
'toString' in {}                              // → true

// instanceof：原型链检查
function User() {}
const u = new User()
u instanceof User                             // → true
```

## 可选链 `?.`（ES2020）

```js
const user = { profile: null }

user.profile.name                             // ✗ TypeError
user.profile?.name                            // → undefined   短路返回

// 函数调用
user.getName?.()                              // 不存在则返回 undefined

// 数组访问
arr?.[0]
```

短路语义：一旦 `?.` 左侧是 `null` / `undefined`，**整条链路立即终止**并返回 `undefined`：

```js
user?.profile.name                            // 若 user 是 nullish，整体 undefined
                                               // 若 user 存在但 profile 是 null，仍 TypeError
```

## 展开 / 剩余 `...`

```js
// 展开（spread）
const a = [1, 2]
const b = [...a, 3, 4]                        // → [1, 2, 3, 4]
const o1 = { x: 1 }
const o2 = { ...o1, y: 2 }                    // → { x: 1, y: 2 }

Math.max(...[3, 1, 4, 1, 5])                  // → 5

// 剩余（rest）
function sum(...nums) { return nums.reduce((a, b) => a + b, 0) }
sum(1, 2, 3, 4)                               // → 10

const [first, ...rest] = [1, 2, 3]
const { a, ...others } = { a: 1, b: 2, c: 3 }
```

`...` 用在右侧字面量是 spread；用在左侧形参或解构是 rest。

## 运算符优先级

JavaScript 运算符优先级共 19 级。常用顺序（**高 → 低**）：

```text
()  []  .  ?.                       成员访问、调用
new
++  --                              后置自增/减
!  ~  +  -  typeof  void  delete    一元
**                                  幂（右结合）
*  /  %                             乘除模
+  -                                加减
<<  >>  >>>                         位移
<  <=  >  >=  in  instanceof
==  !=  ===  !==
&  ^  |
&&
||  ??
?:                                  三元
=  +=  -=  ...                      赋值（右结合）
,                                   逗号
```

不确定时**加括号**，永远是更安全的选择：

```js
const x = a + b * c                           // 等价于 a + (b * c)
const y = (a + b) * c

// 易错：?? 与 || 必须括号
const v = a ?? b || c                         // ✗ SyntaxError
const v2 = (a ?? b) || c                      // ✓
```

## 短路求值的副作用

```js
// 仅在前提成立时执行后置操作
isLoggedIn && trackEvent('view')

// 安全访问 + 默认
const port = config?.server?.port ?? 8080

// 提前 return（注意可读性）
if (!user) return null
const name = user.name
```

## 小结

- JavaScript 运算符种类繁多，但落到日常代码主要是算术、逻辑、赋值、解构、可选链、展开六类；
- `||` 与 `??` 的差异是工程中高频踩坑点：默认值场景几乎总是 `??` 更合适；
- 模板字符串、解构、`?.`、`??` 是 ES6+ 引入的人体工学利器，能显著简化代码；
- 不确定优先级时加括号，可读性 > 简洁性。
