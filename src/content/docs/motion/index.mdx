# 动画系列导读

本系列覆盖 React 下最主流的两套动画方案——**Motion for React**（原 Framer Motion）和 **GSAP**——从零基础到工程实战。

<Markmap markdown={motionTree} />

## 为什么把这两个放在一起

| 方案 | 心智模型 | 包大小 | 强项 |
| ---- | -------- | ------ | ---- |
| Motion for React | 声明式、组件化 | ~40 KB | 业务交互、布局动画、手势、滚动 |
| GSAP | 命令式、时间轴 | ~50 KB（按需） | 复杂时间轴、SVG、营销动效、滚动叙事 |

两者**不冲突**——同一个项目里通常用 Motion 写业务交互（按钮、Modal、列表），用 GSAP 写需要精细时间轴的 Hero / 滚动故事段落。本系列分别讲透，最后给出选型与混用方案。

## 阅读路径

### 零基础

按章节顺序读。前 3 章先把 CSS 动画、`requestAnimationFrame`、合成层这些前置知识补齐，再进入 Motion。

### 有一定 React 经验

跳过第 1、2 章，从 [如何选型](/motion/choosing) 开始。Motion 入门后可直接看 [布局动画](/motion/motion-layout) 和 [Variants 编排](/motion/motion-variants) ——这两个是 Motion 的杀手锏。

### 来做营销页 / 滚动叙事

直接跳到 GSAP 篇的 [ScrollTrigger](/motion/gsap-scroll-trigger) 与 [SplitText](/motion/gsap-split-text)。
