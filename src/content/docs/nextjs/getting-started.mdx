# 快速起步

从零到一个可运行的 Next.js 16 项目。

## 环境要求

- **Node.js 20+**（推荐 22 LTS）
- **pnpm** / npm / yarn / bun（本文使用 pnpm）

```bash
node -v   # ≥ 20
pnpm -v   # ≥ 9
```

## 创建项目

```bash
npx create-next-app@latest my-app
```

交互式选项（推荐配置）：

```text
√ Would you like to use TypeScript?        Yes
√ Would you like to use ESLint?            Yes
√ Would you like to use Tailwind CSS?      Yes
√ Would you like to use `src/` directory?  Yes
√ Would you like to use App Router?        Yes
√ Would you like to customize the default
  import alias (@/*)?                      Yes
√ Would you like to use Turbopack?         Yes (Next.js 16 默认)
```

## 三个核心命令

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev --turbo",
    "build": "next build",
    "start": "next start"
  }
}
```

| 命令 | 用途 | Turbo 支持 |
| ---- | ---- | ---------- |
| `pnpm dev` | 开发服务器，HMR 热更新 | 默认 `--turbo` |
| `pnpm build` | 生产构建 | Next.js 16 默认 Turbo |
| `pnpm start` | 启动生产服务器 | 不适用（已构建完成） |

```bash
pnpm dev      # http://localhost:3000
```

## 初始目录结构

```text
my-app/
├── src/
│   └── app/
│       ├── favicon.ico
│       ├── globals.css
│       ├── layout.tsx          # 根布局（必需）
│       └── page.tsx            # 首页 /
├── public/                     # 静态资源（原样输出）
├── next.config.ts              # Next.js 配置
├── package.json
├── tsconfig.json
└── postcss.config.mjs          # Tailwind CSS 配置
```

## 理解关键文件

### `app/layout.tsx` —— 根布局

```tsx filename="src/app/layout.tsx"
import type { Metadata } from 'next'
import './globals.css'

export const metadata: Metadata = {
  title: 'My Next.js App',
  description: 'Generated by create-next-app',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="zh-CN">
      <body className="antialiased">{children}</body>
    </html>
  )
}
```

要点：
- **必须有根布局**：每个 App Router 项目必须有顶层的 `app/layout.tsx`；
- **必须包含 `<html>` 和 `<body>`**：根布局负责 HTML 骨架；
- **`<html lang>` 属性在此设置**：不走 `_document.tsx`（那是 Pages Router）;
- **`metadata` 导出**：根级 SEO 信息，子页面可覆盖。

### `app/page.tsx` —— 首页

```tsx filename="src/app/page.tsx"
export default function Home() {
  return (
    <main className="flex min-h-screen items-center justify-center">
      <h1 className="text-3xl font-bold">Hello, Next.js 16!</h1>
    </main>
  )
}
```

文件路径决定路由：`app/page.tsx` → `/`，`app/about/page.tsx` → `/about`。

### `next.config.ts` —— 配置

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  // 默认无需配置
}

export default nextConfig
```

详见 [配置体系](/nextjs/configuration)。

## 添加一个新页面

```tsx filename="src/app/about/page.tsx"
import Link from 'next/link'

export default function About() {
  return (
    <main>
      <h1>About</h1>
      <Link href="/">← Back home</Link>
    </main>
  )
}
```

访问 `http://localhost:3000/about` 即可看到。不需要配置路由——**目录结构就是路由**。

## 开发服务器行为

```text
┌──────────────────────────────────────────────────────────┐
│                      next dev --turbo                     │
│                                                           │
│  • Turbopack 编译，HMR < 100ms                            │
│  • 路由按需编译（首次访问 /about 时才编译 about/page.tsx） │
│  • 类型检查独立运行（不阻塞编译）                          │
│  • layout 在路由切换时保持状态（不 remount）               │
│  • Webpack 来源的插件不兼容 — 需迁移到 Turbopack 插件     │
└──────────────────────────────────────────────────────────┘
```

## 陷阱

### CJS / ESM 模块混用

某些旧 npm 包是 CommonJS 格式。在 Server Components 中直接 `import` 可能报错：

```tsx
// 报错：require() of ES Module is not supported
import oldLib from 'some-cjs-lib'
```

**修复**：

```tsx
// 动态 import 强制兼容
const oldLib = await import('some-cjs-lib')
```

或在 `next.config.ts` 启用 `serverExternalPackages`：

```ts
const nextConfig: NextConfig = {
  serverExternalPackages: ['some-cjs-lib'],
}
```

### Tailwind v4 语法变化

Next.js 16 项目默认使用 Tailwind CSS v4，不再需要 `tailwind.config.ts`：

```css filename="src/app/globals.css"
@import "tailwindcss";
```

旧的 `tailwind.config.ts` 不会被自动读取，如有自定义 theme 需迁移到 CSS-first 配置。

## 小结

- `create-next-app` 生成标准骨架，Turbo 默认开启；
- `app/page.tsx` 即首页路由，`app/layout.tsx` 为根布局（必需）；
- 添加路由 = 添加文件夹 + `page.tsx`，无需路由器配置；
- `dev` / `build` / `start` 三命令覆盖完整开发周期。
