# 工具链

Vue 3 生态的标准工具链以 **Vite** 为核心，提供极速的开发体验。本章从零搭建一个生产级 Vue 3 项目。

## 快速开始：create-vue

Vue 官方提供 `create-vue` 脚手架，基于 Vite 构建：

```bash
npm create vue@latest
```

交互式选项：

```text
✔ Project name: my-vue-app
✔ Add TypeScript? Yes
✔ Add JSX Support? No
✔ Add Vue Router for Single Page Application development? Yes
✔ Add Pinia for state management? Yes
✔ Add Vitest for Unit Testing? Yes
✔ Add an End-to-End Testing Solution? Playwright
✔ Add ESLint for code quality? Yes
✔ Add Prettier for code formatting? Yes
```

生成的项目结构：

```text
my-vue-app/
├── public/              # 静态资源
├── src/
│   ├── assets/          # 编译资源（图片、样式）
│   ├── components/      # 组件
│   ├── router/          # 路由配置
│   ├── stores/          # Pinia 状态
│   ├── views/           # 页面组件
│   ├── App.vue          # 根组件
│   └── main.ts          # 入口文件
├── index.html           # HTML 模板
├── vite.config.ts       # Vite 配置
├── tsconfig.json        # TypeScript 配置
└── package.json
```

启动开发服务器：

```bash
cd my-vue-app
npm install
npm run dev
```

## Vite：下一代构建工具

Vite 的核心优势：

1. **极速冷启动**：利用浏览器原生 ESM，无需打包
2. **即时热更新**：模块级 HMR，毫秒级响应
3. **按需编译**：只编译当前页面需要的模块
4. **生产优化**：基于 Rollup 打包，自动代码分割

### Vite 配置

```ts filename="vite.config.ts"
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [
    vue()
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server: {
    port: 3000,
    open: true
  },
  build: {
    target: 'esnext',
    sourcemap: true
  }
})
```

常用插件：

```bash
# 自动导入 API
npm i -D unplugin-auto-import unplugin-vue-components

# 自动导入图标
npm i -D unplugin-icons

# SVG 组件化
npm i -D vite-svg-loader
```

```ts filename="vite.config.ts"
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import Icons from 'unplugin-icons/vite'
import IconsResolver from 'unplugin-icons/resolver'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      imports: ['vue', 'vue-router', 'pinia'],
      dts: 'src/auto-imports.d.ts'
    }),
    Components({
      resolvers: [IconsResolver()],
      dts: 'src/components.d.ts'
    }),
    Icons({ autoInstall: true })
  ]
})
```

## TypeScript 集成

Vue 3 对 TypeScript 有一流支持。

### 基础配置

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "jsx": "preserve",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "esModuleInterop": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "skipLibCheck": true,
    "types": ["vite/client"],
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src/**/*.ts", "src/**/*.vue"],
  "exclude": ["node_modules"]
}
```

### 组件类型定义

```vue filename="components/Button.vue"
<script setup lang="ts">
interface Props {
  label: string
  variant?: 'primary' | 'secondary'
  disabled?: boolean
}

const props = withDefaults(defineProps<Props>(), {
  variant: 'primary',
  disabled: false
})

const emit = defineEmits<{
  click: [event: MouseEvent]
}>()
</script>

<template>
  <button 
    :class="['btn', `btn-${variant}`]"
    :disabled="disabled"
    @click="emit('click', $event)"
  >
    {{ label }}
  </button>
</template>
```

### Volar：Vue 的 TypeScript 支持

安装 VS Code 扩展：

- **Vue - Official**（原 Volar）：Vue 3 语法高亮、类型检查
- **TypeScript Vue Plugin**：`.vue` 文件的 TS 支持

配置 `tsconfig.json`：

```json
{
  "compilerOptions": {
    "plugins": [
      {
        "name": "@vue/typescript-plugin"
      }
    ]
  }
}
```

## 环境变量

Vite 使用 `.env` 文件管理环境变量：

```bash filename=".env"
VITE_APP_TITLE=My Vue App
VITE_API_BASE_URL=https://api.example.com
```

```bash filename=".env.development"
VITE_API_BASE_URL=http://localhost:3000
```

```bash filename=".env.production"
VITE_API_BASE_URL=https://api.production.com
```

在代码中使用：

```ts filename="src/config.ts"
export const config = {
  appTitle: import.meta.env.VITE_APP_TITLE,
  apiBaseUrl: import.meta.env.VITE_API_BASE_URL
}
```

类型定义：

```ts filename="src/env.d.ts"
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_APP_TITLE: string
  readonly VITE_API_BASE_URL: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

## 陷阱与最佳实践

### ❌ 不要在 `<script setup>` 中使用 `export default`

```vue
<!-- ❌ 错误 -->
<script setup>
export default {
  name: 'MyComponent'
}
</script>
```

```vue
<!-- ✅ 正确 -->
<script setup>
// <script setup> 自动导出，无需 export default
</script>
```

### ❌ 不要混用 `<script>` 和 `<script setup>`

```vue
<!-- ❌ 错误 -->
<script>
export default {
  name: 'MyComponent'
}
</script>

<script setup>
import { ref } from 'vue'
const count = ref(0)
</script>
```

```vue
<!-- ✅ 正确：需要 name 时使用单独的 <script> -->
<script lang="ts">
export default {
  name: 'MyComponent'
}
</script>

<script setup lang="ts">
import { ref } from 'vue'
const count = ref(0)
</script>
```

### ✅ 使用路径别名

```ts filename="vite.config.ts"
resolve: {
  alias: {
    '@': fileURLToPath(new URL('./src', import.meta.url)),
    '@components': fileURLToPath(new URL('./src/components', import.meta.url))
  }
}
```

```vue
<script setup>
import Button from '@components/Button.vue'
</script>
```
