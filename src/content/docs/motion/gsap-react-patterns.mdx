# GSAP React 协作模式

这一章把前面散布在各章里的"GSAP + React 协作要点"汇总成一份完整指南——是写出**健壮、可维护、零内存泄漏**GSAP React 代码的清单。

## 核心：用 `useGSAP`

只要在 React 里写 GSAP，就用 `useGSAP`——这是不容妥协的规则。它解决了：

- StrictMode 的双调用
- 组件卸载时的 cleanup
- SSR 环境的服务端报错
- 选择器范围控制

```tsx
import { useGSAP } from '@gsap/react'
import gsap from 'gsap'
import { useRef } from 'react'

function Component() {
  const container = useRef<HTMLDivElement>(null)

  useGSAP(
    () => {
      // 所有 GSAP 代码放这里
    },
    { scope: container }
  )

  return <div ref={container}>...</div>
}
```

## 模式 1：进场动画

最常见——组件挂载时播放一次：

```tsx
useGSAP(
  () => {
    gsap.from('.title', { y: 30, opacity: 0, duration: 0.6 })
    gsap.from('.btn', { scale: 0, duration: 0.4, delay: 0.3 })
  },
  { scope: container }
)
```

`useGSAP` 默认依赖 `[]`——只在 mount 时跑一次。组件卸载时自动 revert。

## 模式 2：响应 state 变化

`dependencies` 控制何时重跑：

```tsx
const [count, setCount] = useState(0)

useGSAP(
  () => {
    gsap.to('.bar', { width: count + '%', duration: 0.4 })
  },
  { scope: container, dependencies: [count] }
)
```

`count` 变化时旧动画被 revert，回调重新执行——和 `useEffect` 一致。

## 模式 3：保存 timeline 引用

需要在事件回调里 play / reverse / seek：

```tsx
const tlRef = useRef<gsap.core.Timeline | null>(null)

useGSAP(
  () => {
    tlRef.current = gsap.timeline({ paused: true })
    tlRef.current
      .from('.title', { y: 30, opacity: 0 })
      .from('.btn', { scale: 0 })
  },
  { scope: container }
)

return (
  <div ref={container}>
    <button onClick={() => tlRef.current?.play()}>播</button>
    <button onClick={() => tlRef.current?.reverse()}>倒</button>
  </div>
)
```

## 模式 4：事件回调里调 GSAP（contextSafe）

事件回调里**直接调用 GSAP** 不会被 `useGSAP` 的 context 收集——卸载时不清理。包一层 `contextSafe`：

```tsx
const { contextSafe } = useGSAP({ scope: container })

const handleClick = contextSafe(() => {
  gsap.to('.box', { x: 100 })
})

return <button onClick={handleClick}>Click</button>
```

或在 `useGSAP` 回调内拿 `contextSafe`：

```tsx
useGSAP(
  (context, contextSafe) => {
    const onClick = contextSafe!(() => {
      gsap.to('.box', { x: 100 })
    })
    const el = container.current?.querySelector('button')
    el?.addEventListener('click', onClick)
    return () => el?.removeEventListener('click', onClick)
  },
  { scope: container }
)
```

> **首选用 React 事件**（`onClick={contextSafe(...)}`），别在 useGSAP 里 addEventListener。

## 模式 5：选择器 vs ref

GSAP 用选择器最直观：

```tsx
gsap.to('.box', { x: 100 })
```

但选择器**默认是全局**——必须配合 `scope` 限定范围：

```tsx
useGSAP(() => {
  gsap.to('.box', { x: 100 })  // 只在 scope 内查找
}, { scope: container })
```

也可以直接给元素 ref：

```tsx
const boxRef = useRef<HTMLDivElement>(null)

useGSAP(() => {
  gsap.to(boxRef.current, { x: 100 })
}, { scope: container })
```

| 场景 | 推荐 |
| ---- | ---- |
| 多个同类元素 | 选择器 + scope |
| 单个动态元素 | useRef |
| 列表渲染的项 | 用 `gsap.utils.toArray` |

### gsap.utils.toArray

```tsx
const items = gsap.utils.toArray<HTMLElement>('.item')
items.forEach((item, i) => {
  gsap.from(item, { y: 30, delay: i * 0.1 })
})
```

`toArray` 把选择器转成真数组——比 `document.querySelectorAll` 更友好。

## 模式 6：列表 + 状态

列表项数量变化时怎么动画？

```tsx filename="components/AnimatedList.tsx"
const [items, setItems] = useState<string[]>(['A', 'B', 'C'])

useGSAP(
  () => {
    gsap.from('.item', { opacity: 0, y: 20, stagger: 0.05 })
  },
  { scope: container, dependencies: [items.length] }
)

return (
  <ul ref={container}>
    {items.map(item => <li key={item} className="item">{item}</li>)}
  </ul>
)
```

`dependencies: [items.length]` 让长度变化时重跑入场动画。

**复杂的增删动画请用 Motion `<AnimatePresence>` + `layout`**——GSAP 单独处理"卸载前播退场"很麻烦。

## 模式 7：清理副本

某些 GSAP 工具（SplitText / ScrollTrigger 等）不会被 `useGSAP` 自动清理：

```tsx
useGSAP(
  () => {
    const split = new SplitText('.title', { type: 'chars' })
    gsap.from(split.chars, { y: 30, stagger: 0.05 })

    return () => split.revert()  // 手动清理
  },
  { scope: container }
)
```

| 工具 | 自动清理 | 需手动 |
| ---- | -------- | ------ |
| tween / timeline | ⭐ | |
| ScrollTrigger 通过 tween 创建 | ⭐ | |
| 独立 `ScrollTrigger.create({...})` | 部分 | 建议手动 kill |
| SplitText | ❌ | 必须 `revert()` |
| Draggable | 部分 | 复杂场景 `kill()` |
| GSDevTools | ❌ | `kill()` |

## 模式 8：SSR（Next.js / Astro）

GSAP 主体能在服务端 import（不访问 window），但**插件**通常会访问 window——必须客户端动态导入或在 Client Component：

```tsx filename="components/ScrollAnim.tsx (Next.js App Router)"
'use client'

import { useGSAP } from '@gsap/react'
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'
import { useRef } from 'react'

gsap.registerPlugin(ScrollTrigger)

export function ScrollAnim() {
  const container = useRef<HTMLDivElement>(null)
  useGSAP(() => { /* ... */ }, { scope: container })
  return <div ref={container}>...</div>
}
```

`'use client'` 让整个文件**只在客户端运行**。

### Astro

```astro
---
import { ScrollAnim } from '../components/ScrollAnim'
---

<ScrollAnim client:load />
```

用 `client:load` / `client:visible` 触发水合。

### 注册插件的位置

`gsap.registerPlugin(...)` 写在哪里？两种风格：

**风格 A**：每个使用插件的文件顶部注册。

```tsx
// hero.tsx
gsap.registerPlugin(ScrollTrigger, SplitText)
```

简单，但同个插件可能被注册多次（幂等，没坏处）。

**风格 B**：全局 setup 文件统一注册。

```tsx filename="lib/gsap-setup.ts"
'use client'
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'
import { SplitText } from 'gsap/SplitText'

if (typeof window !== 'undefined') {
  gsap.registerPlugin(ScrollTrigger, SplitText)
}
```

```tsx filename="app/layout.tsx"
import '../lib/gsap-setup'
```

工程化但要小心 SSR 边界。

## 模式 9：StrictMode 已解决

React 18+ 开发模式下 `useEffect` 故意双调用——**`useGSAP` 内部用 `gsap.context()` 收集动画**，第二次调用前自动 revert 第一次。**不用做任何事**，写普通代码即可。

## 模式 10：与 Motion 共存

同一项目里 Motion 处理业务、GSAP 处理叙事——前提是**不要让两个库管同一个元素**：

```tsx
{/* ❌ GSAP 和 Motion 都在改 .box 的 transform */}
<motion.div className="box" animate={{ x: 100 }} />

useGSAP(() => {
  gsap.to('.box', { x: 100 })   // 互相覆盖
}, { scope: container })
```

```tsx
{/* ✅ 各管各的 */}
<motion.div className="motion-managed" animate={{ x: 100 }} />

useGSAP(() => {
  gsap.to('.gsap-managed', { x: 100 })
}, { scope: container })
```

## 完整模板：可拷贝即用

```tsx filename="components/AnimatedSection.tsx"
'use client'  // Next.js / Astro 需要

import { useGSAP } from '@gsap/react'
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'
import { useRef } from 'react'

gsap.registerPlugin(ScrollTrigger)

interface Props {
  children: React.ReactNode
}

export function AnimatedSection({ children }: Props) {
  const container = useRef<HTMLElement>(null)
  const tlRef = useRef<gsap.core.Timeline | null>(null)

  const { contextSafe } = useGSAP(
    () => {
      // 1. 创建主 timeline
      tlRef.current = gsap.timeline({
        defaults: { ease: 'power2.out' },
        scrollTrigger: { trigger: container.current, start: 'top 80%' },
      })

      tlRef.current
        .from('.title', { y: 30, opacity: 0, duration: 0.6 })
        .from('.subtitle', { y: 20, opacity: 0, duration: 0.5 }, '-=0.3')

      // 2. 清理 ScrollTrigger（保险起见）
      return () => ScrollTrigger.refresh()
    },
    { scope: container }
  )

  // 3. 事件回调用 contextSafe
  const handleReplay = contextSafe(() => {
    tlRef.current?.restart()
  })

  return (
    <section ref={container}>
      <h2 className="title">标题</h2>
      <p className="subtitle">副标题</p>
      {children}
      <button onClick={handleReplay}>重播</button>
    </section>
  )
}
```

这一个模板包含：

- `'use client'` 标记
- 插件注册
- ref 与 scope
- timeline 引用保存
- ScrollTrigger 配合
- contextSafe 事件回调
- 类型完整

新写组件时直接复制 + 改细节。

## 陷阱速查

| 现象 | 可能原因 |
| ---- | -------- |
| 动画播两次 | 没用 `useGSAP`，被 StrictMode 双调用 |
| 动画跑错元素 | 没传 `scope`，选择器全局命中 |
| 卸载后 console 出 GSAP 警告 | 事件回调里直接调 GSAP，没用 `contextSafe` |
| SplitText 嵌套 DOM 越来越多 | 没在 cleanup 里 `split.revert()` |
| 路由切换后 ScrollTrigger 错位 | 切完后没调 `ScrollTrigger.refresh()` |
| SSR 报 `window is not defined` | 插件没在客户端边界内导入 |
| state 变化但动画不重跑 | `useGSAP` 没加 `dependencies` |

## 小结

- 永远用 `useGSAP` 包 GSAP 代码——它处理 StrictMode、cleanup、SSR
- 永远传 `scope`——避免选择器全局污染
- 事件回调里调 GSAP 要用 `contextSafe`
- SplitText / GSDevTools 等需要在 return 函数里手动 revert / kill
- SSR 框架中插件代码放进 Client Component
- 与 Motion 共存时确保两库不管同一个元素
- 拷贝本章末尾的"完整模板"作为新组件起点
