# Material 3 与 Cupertino

Flutter 内置两套系统级 UI 套件：

- **Material**：Google 设计语言，对齐 Android 默认外观；最新版本是 **Material 3**（M3，自 Flutter 3.16 起为默认）。
- **Cupertino**：iOS 风格组件，外观与 iOS 系统应用一致。

业务里 95% 项目以 **Material 3 为主，关键路径补 Cupertino**——这也是本章的取舍。

## 启动 Material 3

```dart
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'My App',
      theme: ThemeData(
        useMaterial3: true,                              // ★ 启用 M3
        colorSchemeSeed: Colors.indigo,                  // 由一个种子色派生整套色板
        brightness: Brightness.light,
      ),
      darkTheme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: Colors.indigo,
        brightness: Brightness.dark,
      ),
      themeMode: ThemeMode.system,                       // 跟随系统
      home: const HomePage(),
    );
  }
}
```

`useMaterial3` 在 Flutter 3.16+ 默认是 `true`，但显式写出能让代码意图清晰。

## ColorScheme：M3 的色彩骨架

M3 抛弃了旧的"primary / accent / 多个 surface 色"散乱命名，改用一套**配对色**结构：

| 角色 | 用途 |
| ---- | ---- |
| `primary` / `onPrimary` | 主色 / 主色之上的前景色 |
| `primaryContainer` / `onPrimaryContainer` | 主色的弱化容器 / 其上前景 |
| `secondary` / `onSecondary` / `secondaryContainer` / `onSecondaryContainer` | 次色 |
| `tertiary` / `onTertiary` / `tertiaryContainer` / `onTertiaryContainer` | 第三色（强调/装饰） |
| `error` / `onError` / `errorContainer` / `onErrorContainer` | 错误色 |
| `surface` / `onSurface` | 卡片、对话框、菜单等表面 |
| `surfaceContainerLowest` / `surfaceContainerLow` / `surfaceContainer` / `surfaceContainerHigh` / `surfaceContainerHighest` | M3 新增的"层次表面"，替代旧 `background` |
| `outline` / `outlineVariant` | 边框、分割线 |

> **Material 3.22 起 `background` / `onBackground` / `surfaceVariant` 被废弃**，换成 `surface*` 一族。新代码不要再使用 `background` 字段。

### 三种获取 scheme 的方式

```dart
// 1. 从种子色生成
final scheme = ColorScheme.fromSeed(seedColor: Colors.indigo);

// 2. 完全手写
const scheme2 = ColorScheme(
  brightness: Brightness.light,
  primary: Color(0xFF6750A4),
  onPrimary: Colors.white,
  // ... 全套填完
);

// 3. 从图片提取（dynamic_color 包，结合系统主题色）
import 'package:dynamic_color/dynamic_color.dart';
DynamicColorBuilder(
  builder: (lightDynamic, darkDynamic) => MaterialApp(
    theme: ThemeData(colorScheme: lightDynamic ?? defaultLight, useMaterial3: true),
    darkTheme: ThemeData(colorScheme: darkDynamic ?? defaultDark, useMaterial3: true),
  ),
);
```

种子色生成是 M3 的精髓——给一个色，算出符合无障碍对比度要求的整套色板。

### 在 widget 里取色

```dart
final scheme = Theme.of(context).colorScheme;
return Container(
  color: scheme.primaryContainer,
  child: Text('hi', style: TextStyle(color: scheme.onPrimaryContainer)),
);
```

**永远从 `Theme.of(context).colorScheme` 取色**，而不是硬编码 `Colors.blue`。这是支持 dark mode 与多主题的前提。

## TextTheme：字体与文本样式

```dart
final t = Theme.of(context).textTheme;
Text('标题', style: t.headlineMedium);
Text('正文', style: t.bodyLarge);
Text('注释', style: t.labelSmall?.copyWith(color: scheme.outline));
```

M3 的 TextTheme 角色：

- **Display**：超大标题（splash、营销）—— `displayLarge` / `Medium` / `Small`
- **Headline**：屏幕标题 —— `headlineLarge` / `Medium` / `Small`
- **Title**：组件标题（卡片标题、对话框标题）—— `titleLarge` / `Medium` / `Small`
- **Body**：正文 —— `bodyLarge` / `Medium` / `Small`
- **Label**：按钮、输入框 label —— `labelLarge` / `Medium` / `Small`

在 ThemeData 里覆写：

```dart
ThemeData(
  useMaterial3: true,
  colorSchemeSeed: Colors.indigo,
  textTheme: const TextTheme(
    bodyLarge: TextStyle(fontSize: 16, height: 1.5),
    headlineMedium: TextStyle(fontWeight: FontWeight.bold),
  ),
);
```

## Scaffold：屏幕脚手架

`Scaffold` 是几乎每个 page 的根 widget，提供 AppBar、body、抽屉、底栏、悬浮按钮、SnackBar 容器。

```dart
Scaffold(
  appBar: AppBar(
    title: const Text('Home'),
    centerTitle: false,
    actions: [
      IconButton(icon: const Icon(Icons.search), onPressed: () {}),
      IconButton(icon: const Icon(Icons.more_vert), onPressed: () {}),
    ],
  ),
  drawer: const NavigationDrawer(children: []),
  body: const Center(child: Text('Body')),
  floatingActionButton: FloatingActionButton(
    onPressed: () {},
    child: const Icon(Icons.add),
  ),
  bottomNavigationBar: NavigationBar(
    selectedIndex: 0,
    onDestinationSelected: (i) {},
    destinations: const [
      NavigationDestination(icon: Icon(Icons.home), label: 'Home'),
      NavigationDestination(icon: Icon(Icons.person), label: 'Me'),
    ],
  ),
);
```

要点：

- `body` 是主内容区，自动避开 AppBar / 底栏 / 键盘安全区；
- **每个 page 一个 Scaffold**；嵌套 Scaffold 仅在你确实想"局部布局含 AppBar"时才用；
- `Scaffold.of(context)` / `ScaffoldMessenger.of(context)` 提供 SnackBar、底栏、Drawer 操作 API。

### `AppBar` vs `SliverAppBar`

```dart
// 普通：固定高度
AppBar(title: const Text('Home'))

// Sliver 版本：嵌入 CustomScrollView，可滚动隐藏 / 折叠
CustomScrollView(
  slivers: [
    SliverAppBar(
      pinned: true,
      floating: false,
      expandedHeight: 200,
      flexibleSpace: const FlexibleSpaceBar(title: Text('Big')),
    ),
    SliverList(...),
  ],
);
```

需要"滚动时 AppBar 折叠 / 视差"用 SliverAppBar；其他用普通 AppBar。

### NavigationBar vs BottomNavigationBar

M3 的底部导航是 `NavigationBar`（5 项以下推荐），旧的 `BottomNavigationBar` 仍可用但属于 M2 风格。

### NavigationRail（侧栏）

平板 / 桌面端用 `NavigationRail` 替代底栏：

```dart
Row(
  children: [
    NavigationRail(
      selectedIndex: 0,
      onDestinationSelected: (i) {},
      destinations: const [
        NavigationRailDestination(icon: Icon(Icons.home), label: Text('Home')),
        NavigationRailDestination(icon: Icon(Icons.person), label: Text('Me')),
      ],
    ),
    const VerticalDivider(width: 1),
    Expanded(child: Body()),
  ],
);
```

响应式切换底栏 / 侧栏的做法见 [布局、约束与主题](/flutter/layout-styling)。

## 按钮全家桶

M3 提供 5 种按钮，按"视觉强度"递减：

| Widget | 用途 | 视觉 |
| ------ | ---- | ---- |
| `FilledButton` | 主要 CTA | 实心、强调色 |
| `FilledButton.tonal` | 次要 CTA | 弱调色 |
| `ElevatedButton` | 浅色背景上的强调 | 带阴影实心 |
| `OutlinedButton` | 边框按钮 | 边框 + 文字 |
| `TextButton` | 文本链接 | 仅文字 |

```dart
FilledButton(onPressed: () {}, child: const Text('Save'))
FilledButton.tonal(onPressed: () {}, child: const Text('Cancel'))
ElevatedButton(onPressed: () {}, child: const Text('Submit'))
OutlinedButton(onPressed: () {}, child: const Text('Skip'))
TextButton(onPressed: () {}, child: const Text('Learn more'))

// 带图标
FilledButton.icon(
  onPressed: () {},
  icon: const Icon(Icons.add),
  label: const Text('Add'),
);

// 圆形纯图标
IconButton.filled(onPressed: () {}, icon: const Icon(Icons.bolt));
IconButton.filledTonal(onPressed: () {}, icon: const Icon(Icons.bolt));
IconButton.outlined(onPressed: () {}, icon: const Icon(Icons.bolt));
IconButton(onPressed: () {}, icon: const Icon(Icons.bolt));
```

`onPressed` 传 `null` 会让按钮**自动进入 disabled 视觉**——这是 Flutter 与命令式框架的关键差异。

### 自定义按钮样式

```dart
FilledButton(
  style: FilledButton.styleFrom(
    minimumSize: const Size(double.infinity, 48),
    padding: const EdgeInsets.symmetric(horizontal: 24),
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(8)),
  ),
  onPressed: () {},
  child: const Text('Continue'),
);
```

更深的样式覆盖通过 `WidgetStateProperty` / `WidgetStatesController`（M3 起 `MaterialState*` 改名为 `WidgetState*`）：

```dart
ElevatedButton(
  style: ButtonStyle(
    backgroundColor: WidgetStateProperty.resolveWith((states) {
      if (states.contains(WidgetState.disabled)) return Colors.grey;
      if (states.contains(WidgetState.pressed)) return Colors.indigo.shade700;
      return Colors.indigo;
    }),
  ),
  onPressed: () {},
  child: const Text('Custom'),
);
```

## Cupertino：iOS 风格组件

需要 iOS 原生即视感时切到 Cupertino。常用组件名以 `Cupertino` 前缀：

| Material | Cupertino |
| -------- | --------- |
| `MaterialApp` | `CupertinoApp` |
| `Scaffold` | `CupertinoPageScaffold` |
| `AppBar` | `CupertinoNavigationBar` |
| `ElevatedButton` | `CupertinoButton.filled` |
| `TextButton` | `CupertinoButton` |
| `TextField` | `CupertinoTextField` |
| `Switch` | `CupertinoSwitch` |
| `Slider` | `CupertinoSlider` |
| `AlertDialog` | `CupertinoAlertDialog` |
| `BottomSheet` | `CupertinoActionSheet` |
| `BottomNavigationBar` | `CupertinoTabBar` |
| 进度 | `CupertinoActivityIndicator` |

完整 iOS 风格的 app：

```dart
CupertinoApp(
  theme: const CupertinoThemeData(brightness: Brightness.light),
  home: CupertinoTabScaffold(
    tabBar: CupertinoTabBar(items: const [
      BottomNavigationBarItem(icon: Icon(CupertinoIcons.home), label: 'Home'),
      BottomNavigationBarItem(icon: Icon(CupertinoIcons.person), label: 'Me'),
    ]),
    tabBuilder: (_, i) => CupertinoPageScaffold(
      navigationBar: CupertinoNavigationBar(middle: Text('Tab $i')),
      child: const Center(child: Text('content')),
    ),
  ),
);
```

### 自适应：单代码两风格

`Switch.adaptive` / `CircularProgressIndicator.adaptive` / `Slider.adaptive` 等会**根据平台自动选择 Material 或 Cupertino 外观**：

```dart
const CircularProgressIndicator.adaptive();   // iOS 上是 Cupertino 风，其他平台是 Material 风
```

不是所有 Material 组件都有 `.adaptive`。需要更细粒度的"按平台分发"用 `Theme.of(context).platform` 或 `defaultTargetPlatform` + 条件构造，详见 [平台集成](/flutter/platform-integration)。

### 何时该用 Cupertino？

实战经验：

- **完全 iOS-only 应用**：直接 CupertinoApp 主导；
- **跨平台主流应用**：以 Material 3 为主，少量关键交互（如 iOS 拨号风格的 Picker）用 Cupertino；
- **强烈"一致风格"诉求**：全 Material 不混 Cupertino，简单直接。

## 主题化最佳实践

### 一处定义，处处引用

```dart
// ✗ 硬编码
Container(color: Colors.indigo)

// ✓ 引用 colorScheme
Container(color: Theme.of(context).colorScheme.primary)
```

### 自定义组件主题

ThemeData 里几乎每个 widget 都有专属主题字段：`appBarTheme` / `cardTheme` / `elevatedButtonTheme` / `inputDecorationTheme` / `textTheme` 等。**集中定义比每处单独传 style 好维护**：

```dart
ThemeData(
  useMaterial3: true,
  colorSchemeSeed: Colors.indigo,
  appBarTheme: const AppBarTheme(centerTitle: false, scrolledUnderElevation: 1),
  inputDecorationTheme: const InputDecorationTheme(
    border: OutlineInputBorder(),
    filled: true,
  ),
  cardTheme: const CardTheme(
    elevation: 0,
    margin: EdgeInsets.zero,
  ),
  filledButtonTheme: FilledButtonThemeData(
    style: FilledButton.styleFrom(minimumSize: const Size.fromHeight(48)),
  ),
);
```

### 扩展自定义颜色（M3 自定义角色）

`ColorScheme` 字段是固定的。要加 brand 专属色（如 `success` / `warning`）用 ThemeExtension：

```dart
@immutable
class AppColors extends ThemeExtension<AppColors> {
  const AppColors({required this.success, required this.warning});

  final Color success;
  final Color warning;

  @override
  AppColors copyWith({Color? success, Color? warning}) =>
      AppColors(success: success ?? this.success, warning: warning ?? this.warning);

  @override
  AppColors lerp(ThemeExtension<AppColors>? other, double t) {
    if (other is! AppColors) return this;
    return AppColors(
      success: Color.lerp(success, other.success, t)!,
      warning: Color.lerp(warning, other.warning, t)!,
    );
  }
}

// 注册
ThemeData(
  useMaterial3: true,
  colorSchemeSeed: Colors.indigo,
  extensions: const [
    AppColors(success: Color(0xFF2E7D32), warning: Color(0xFFEF6C00)),
  ],
);

// 使用
final colors = Theme.of(context).extension<AppColors>()!;
Container(color: colors.success);
```

## Material 3 与 Cupertino 心智模型清单

1. **`useMaterial3: true` + `colorSchemeSeed`** 是 M3 应用的最简启动姿势。
2. **永远从 `Theme.of(context).colorScheme` 取色**，不硬编码 `Colors.xxx`；这是支持多主题的前提。
3. **背景色用 `surface*` 一族**，`background` 已废弃。
4. **每个 page 一个 Scaffold**，AppBar / 底栏 / FAB 全靠它统一管理。
5. **按钮 5 兄弟按视觉强度选**：FilledButton（主 CTA）→ tonal → Elevated → Outlined → Text。
6. **`onPressed: null` 自动 disabled**——别自己写 if 切样式。
7. **SnackBar 用 `ScaffoldMessenger.of(context)`**；旧 `Scaffold.of` 仅用于 BottomSheet / Drawer。
8. **`DropdownMenu` / `SegmentedButton` / `NavigationBar`** 是 M3 替代旧 widget 的新成员。
9. **`.adaptive` 工厂**让一些组件按平台自动选 Material / Cupertino 风。
10. **ThemeExtension** 是 M3 颜色之外的扩展机制，为 brand 色 / 自定义角色服务。
